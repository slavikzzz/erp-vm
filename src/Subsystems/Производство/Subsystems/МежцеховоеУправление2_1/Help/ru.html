Межцеховое управление

Назначение подсистемы

Формирование графика производства предназначено для достижения равномерной загрузки доступных производственных мощностей и преодоления пиковых нагрузок, а также эффективного использования трудовых ресурсов. При формировании графика производства осуществляется управление доступностью рабочих центров, которая может быть ограничена, например, проводимыми ремонтными работами.

Работа подсистемы

При использовании в системе механизма планирования, формирование заказов на производство может быть осуществлено при помощи рабочего места Формирование заказов по плану в количестве, зафиксированном планами производства. Для формирования заказов на производство (поставщику, сборку/разборку, перемещение) с учетом оперативной обстановки на предприятии предназначено рабочее место Формирование заказов по потребности.
Контроль доступных производственных мощностей выполняется в рамках рабочего места Доступность видов рабочих центров, в котором доступность рабочих центров задается в виде значения времени для каждого интервала планирования в рамках рассматриваемого временного периода. Указанное значение показывает максимальное для интервала планирования время, в течение которого рабочий центр может выполнять производственные задания, и не может превышать длительность интервала планирования, для которого задается.

Рабочие центры наследуют от подразделений, в которых находятся, значение интервала планирования, который определяет степень детализации задаваемого графика доступности.

Формирование графика производства осуществляется при помощи установки приоритетов для сформированных заказов на производство, находящихся в статусе К производству. В рамках рабочего места График производства главный диспетчер может контролировать ход выполнения графика производства:

  • оценить исполнимость полученных заказов на производство к дате потребности;
  • оптимизировать объем оборотных средств для обеспечения предприятия ресурсами, использующимися в производстве, приобретая их ближе к датам непосредственного потребления;
  • сформировать производственные программы отдельных подразделений;
  • обеспечить контроль хода производства;
  • оперативно управлять изменениями и отклонениями в производственных планах;
  • оптимизировать потребность в трудовых ресурсах.


Расчет графика может выполняться как для каждого заказа индивидуально, так и командой Планировать очередь до для списка заказов, который определяется от последнего запланированного заказа до произвольно выбранного в очереди заказа.

Общий порядок формирования нового графика производства в ходе перепланирования аналогичен порядку создания графика производства.

Алгоритм размещения этапов при составлении графика производства задается индивидуально для каждой строки заказа на производство параметром Размещение:

  • К началу (используется по умолчанию) – соответствует варианту планирования «как можно быстрее»;
  • К окончанию – соответствует варианту планирования «точно к сроку».

Документ Заказ на производство обеспечивает два режима выполнения производственных процессов:

  • производство под обособленную потребность - возможность использования обособления определяется функциональной опцией НСИ и администрирование - Склад и доставка - Обеспечение потребностей - Обособленное обеспечение заказов;
  • производство к дате потребности - непосредственная связь между строкой заказа на производство как партией выпуска и любым из заказов, затребовавших изделие или работу, отсутствует. Проследить ход производства можно только по партии выпуска, обеспечивающей потребность в продукции, работе на определенную дату. Партии выпуска задаются отдельными строками табличной части документа Заказ на производство.

Анализ данных

Для анализа сформированных графиков производства предназначены отчеты:

  • Загрузка видов рабоих центров по графику производства,
  • Занятость видов рабочих центров продукцией по графикку производства,
  • Исполнение плана производства по подразделениям,
  • Сбалансированность планов.
