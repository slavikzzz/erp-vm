Внутрицеховое управление

Назначение подсистемы

Подсистема предназначена для организации и контроля работ в подразделениях на внутрицеховом уровне. Принятие оперативных решений в подразделениях делегируется локальным диспетчерам.

Работа подсистемы

Для локального диспетчера сформировано рабочее место Диспетчирование производства.

Функции локального диспетчера:

  • формирование для исполнения этапов маршрутных листов;
  • контроль обеспечения выполняемых этапов ресурсами;
  • составление расписания выполняемых работ, загрузки рабочих центров;
  • мониторинг хода выполнения производственных процессов в своем подразделении;
  • перепланирование расписания выполнения маршрутных листов;
  • своевременное оповещение главного диспетчера о критических отклонениях от графика производства.

Диспетчер в рабочем месте в окне Формирование и передача к выполнению маршрутных листов анализирует состав этапов, размещенных к выполнению в его подразделение по интервалам планирования. Формирование документов Маршрутный лист для этапов может выполняться индивидуально для каждого этапа или для списка этапов. На этапе формирования маршрутных листов локальный диспетчер может уточнить параметры их исполнения (например, уточнить потребность в ресурсах или указать исполнителей для видов работ). Вносимые изменения фиксируются на недоступной для редактирования закладке Отклонения от заказа и доступны для анализа.
Подготовленные маршрутные листы передаются к выполнению производственному менеджеру (начальнику смены, мастеру, бригадиру) в статусе К выполнению.
На основании маршрутных листов формируются требования на получение со склада контролируемых по обеспечению материалов (со способом обеспечения Заказ под заказ).
Если маршрутный лист сформирован, но к выполнению не принят (не установлен статус К выполнению или выше), то решение о перепланировании этапа остается на усмотрение главного диспетчера.
Контроль своевременного выполнения подразделениями производственных этапов на основании расписаний установленных для каждого из маршрутных листов в статусах К выполнению, осуществляется при помощи рабочего места Выполнение маршрутных листов.

В отдельном документе отражается Выработка сотрудников.
Факт выпуска материальных результатов производственного процесса, а также работ и услуг отражается документом Выпуск продукции.

Выпуск продукции может быть оформлен и без графика производства. В этом случае факт выпуска отражается в системе документами Выпуск продукции и Выполнение работ и услуг с типом операции Без распоряжения и связан не с интервалами планирования, а с отчетными периодами финансового учета, по которым производится расчет себестоимости. Обеспечение подразделений материальными ресурсами выполняется вручную созданными документами Внутреннее потребление товаров или на основании внутренних заказов.

Пооперационное планирование

Пооперационное планирование возможно в рамках подразделений, для которых в справочнике Структура предприятия параметру Управление маршрутными листами установлено значение Пооперационное планирование.

Пооперационное планирование может выполняться двумя способами:

  • Планирование вперед.
  • Планирование назад.


Для способа Планирование вперед планирование каждого маршрутного листа производится начиная с первой операции. Каждая следующая операция планируется после размещения предыдущей операции, которая уже запланирована.
Для способа Планирование назад планирование каждого маршрутного листа производится начиная с последней операции. Каждая предыдущая операция планируется до начала следующей, которая уже запланирована.
Для обоих способов планирования при размещении операций учитывается время транспортировки между видами рабочих центров Параметры межоперационных переходов (Производство/Настройки и справочники).
Составление расписания и диспетчеризация при пооперационном планировании выполняется в окне Пооперационное планирование в рабочем месте Управление маршрутными листами.
Последовательность размещения операций определяется порядком исполнения маршрутного листа, к которому эти операции относятся. Порядок маршрутных листов определяет локальный диспетчер.

Анализ данных

Для анализа загрузки рабочих центров, сформировавшейся при составлении расписания, предназначен отчет Загрузка рабочих центров. Вариант данного отчета с детализацией до состава выпускаемой продукции получил название Занятость рабочих центров продукцией. Отчет Причины задержек выполнения маршрутных листов предназначен для анализа причин невыполнения/несвоевременного выполнения производственных этапов. Для анализа стабильности выполнения маршрутных листов используются отчеты Выполнение маршрутных листов без задержек и Выполнение маршрутных листов с задержками.
