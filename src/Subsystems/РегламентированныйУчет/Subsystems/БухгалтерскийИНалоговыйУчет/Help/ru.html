Бухгалтерский и налоговый учет

Назначение подсистемы

Подсистема предназначена для отражения в системе различных неавтоматизированных операций осуществляемых в рамках регламентированного учета. При необходимости в Журнале проводок можно отобразить при помощи фильтров нужные проводки с целью проведения аудита.

Работа подсистемы

Рабочее место Отражение документов в регламентированном учете предназначено для настройки гибкой системы отражения в регламентированном учете всех производимых хозяйственных операций (проводок), как в разрезе различных организаций, так и в разрезе различных форм её собственности.

Отражение документов в регламентированном учете осуществляется в два этапа:

  • Настройка счетов учета,
  • Отражение документов в регламентированном учете.

Настройка счетов учета настраивается по группам:

  • Собственная номенклатура,
  • Номенклатура, переданная на комиссию,
  • Расчеты с контрагентами,
  • Прочие доходы,
  • Расходы,
  • Прочие операции.

Счета учета товарно-материальных ценностей настраиваются для склада и группы настроек финансового учета номенклатуры (реквизит справочника Номенклатура). Особое внимание следует обратить на то, что для разных видов финансового учета (РСБУ и МСФО) используются единые группы настроек финансового учета номенклатуры. Состав групп должен изначально определяться с учетом отражения во всех видах финансового учета.
Счета учета расчетов с контрагентами настраиваются для группы финансового учета расчетов, которая является реквизитом документа расчетов.
Счета учета прямых производственных затрат (счета 20, 23, 29) устанавливаются для каждого подразделения. Аналитический учет (субконто) представлен типом затрат (материальные, работы, сдельная оплата труда, прочие) и самой затратой (номенклатура, вид работ сотрудника, статья расходов).
Счета учета косвенных затрат и прочих расходов (счета 25, 26, 44 и 91) настраиваются для подразделения и статьи расходов, счета учета доходов – для подразделения и статьи доходов.
Отражение в регламентированном учете произвольных операций с активами предприятия (например, списание денежных средств на прочие нужды) задается в индивидуальном порядке по каждому документу, которым данная операция была зафиксирована в оперативном учете.

В течение отчетного периода пользователи могут вносить данные в информационную базу в произвольной последовательности. При этом значения сумм, указанные в предварительно сформированных проводках по отдельным документам, могут потерять свою достоверность. В решении автоматически отслеживается возникновение таких ситуаций через механизм актуализации проводок. Для корректного отражения проводок для измененных документов требуется восстановить актуальность их отражения в регламентированном учете.

Механизмы формирования проводок не используют бухгалтерские итоги. Проводки формируются в привязке к первичным документам, что обеспечивает возможность детальной сверки данных бухгалтерских проводок и первичных документов.

Анализ данных

Для контроля полноты отражения хозяйственных операций в регламентированном учете ведется постоянный автоматический мониторинг состояния отражения документов. Корректность отражения хозяйственных операций можно проверить стандартными отчетами регламентированного учета:

  • Оборотно-сальдовая ведомость,
  • Оборотно-сальдовая ведомость по счету,
  • Обороты счета,
  • Обороты между субконто,
  • Анализ счета,
  • Анализ субконто,
  • Карточка счета,
  • Карточка субконто,
  • Отчет по проводкам,
  • Сводные проводки.
