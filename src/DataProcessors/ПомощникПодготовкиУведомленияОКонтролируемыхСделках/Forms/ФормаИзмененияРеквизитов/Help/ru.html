Форма изменения реквизитов

Данная форма представляет собой удобный механизм изменения реквизитов справочников объектов. Форма условно поделена на две части, в левой части отображаются реквизиты, которые доступны для изменения и их значения, в правой же части отображаются доступные для изменения справочники и значения их элементов.

Работа с изменяемыми реквизитами

Изменение редактируемого реквизита

Список реквизитов, над которыми может производиться работа в данной форме, определяется программно и недоступен в пользовательском интерфейсе.

Любой из реквизитов, выбранных разработчиком, доступен пользователю и может редактироваться в поле над списком значений реквизита. При этом если разработчиком выбран в качестве доступного для изменения только один реквизит, это поле будет скрыто.

При выборе реквизита левый список значений обновится данными по этому реквизиту. При этом он будет заполнен всеми значениями, которые может принимать данный реквизит.

Пример: если мы изменяем справочник номенклатура и выбрали для изменения "Код ТНВЭД", список значений заполнится всеми элементами справочника "Классификатор ТНВЭД".

Список значений реквизита

Здесь перечислены все возможные значения, которые может принимать данный реквизит. В случае, если реквизит является справочником - в этом поле будут перечислены все элементы данного справочника, в случае если реквизит булевский - в списке будут находиться только 2 элемента, соответствующие "Истина" и "Ложь".

В крайней правой колонке выведен флажок, который показывает есть ли уже элементы справочника, который мы изменяем, с таким значением реквизита или нет.

Так же, в случае необходимости (предусматривается разработчиком) в крайней левой колонке выведен флажок - специальная пометка данного значения реквизита.

Например, такая пометка может понадобится, чтобы выделить все страны, которые находятся в оффшорной зоне, при настройке стран регистрации контрагентов.

Для реквизита с типом справочник всегда добавляется пустое значение данного справочника, для того, чтобы можно было выбрать значения, в которых изменяемый реквизит не заполнен,

Работа со значениями изменяемого справочника

Изменение редактируемого справочника

Список справочников, над которыми может производиться работа в данной форме, определяется программно и недоступен в пользовательском интерфейсе.

Любой из справочников, выбранных разработчиком, доступен пользователю и может редактироваться в поле над списком значений справочника. При этом если разработчиком выбран в качестве доступного для изменения только один справочник, это поле будет скрыто.

Список значений справочника

Здесь выводятся значения справочника, выбранного для изменения, которые содержат выделенный в данный момент изменяемый реквизит (в левом списке). То есть, когда мы аквтивизируем какую-либо из строк левого списка - мы накладываем отбор на справочник и в нем выбираются только те элементы, в которых изменяемый реквизит равен выбранному значению в левом списке.

Для удобства пользователя, если в левом списке возможных значений реквизита выбран пустой элемент, правый список выводится с иерархией (если изменяемый справочник является иерархическим). Во всех остальных случаях - правый список содержит элементы.

При желании, можно наложить на данный список любой отбор и группировку необходимую пользователю для комфортной работы.

Изменение объектов

Перенос по кнопке

Над правым списком, где содержатся список значений изменяемого справочника есть кнопка Для выделенных элементов и групп перенести в новое значение. Она осуществляет назначение объектам, выделенных в правом списке нового значения реквизита из левого списка.

В правом списке у нас отображаются те значения изменяемого справочника, которые имеют изменяемый реквизит равный выделенному значению левого списка (список значений изменяемого реквизита). По нажатию на данную кнопку открывается список выбора, который по сути дублирует левый список. Здесь выбираем новое значение реквизита и для всех выделенных элементов изменяемого справочника изменяемый реквизит приобретает новое значение, только что выбранное нами.

Данный принцип будет работать и тогда, когда мы выделяем для переноса иерархическую группу значений (перенесутся все элементы, находящиеся в этой группе) или некоторую произвольную группировку (перенесутся все элементы, входящую в эту группировку). Так же, при переносе, будет учитываться отбор пользователя, наложенный на список элементов изменяемого справочника.

Подбор

Для некоторых справочников в системе существуют типовые формы подбора. Если они заданы разработчиком, над правым списком появится кнопка подбора. Она позволяет для текущего значения изменяемого реквизита в левом списке подобрать новые значения изменяемого справочника (помимо тех, что уже представлены в правом списке). При нажатии на нее открывается форма подбора, и мы на основании произвольных отборов можем указать каким элементам необходимо назначить новые значения изменяемого реквизита.

Перетаскивание

В этой форме работает перетаскивание. Выделяем элементы в правом списке, для которых необходимо установить новое значение изменяемого реквизита, и перетаскиваем их в любую строку левого списка. В этом случае изменяемый реквизит выделенных элементов приобретет значение из строки, в которую мы произвели перетаскивание.

Данный принцип будет работать и тогда, когда мы выделяем для переноса иерархическую группу значений (перенесутся все элементы, находящиеся в этой группе) или некоторую произвольную группировку (перенесутся все элементы, входящую в эту группировку). Так же, при переносе, будет учитываться отбор пользователя, наложенный на список элементов изменяемого справочника.

Изменение множества элементов

Если элементов для изменения больше 10, выполнение изменения будет производиться на сервере в фоновом задании. Работа с текущей формой будет заблокировано, а так же появится окошко прогресса выполнения, которое будет отображать сколько элементов уже обработано (в процентах).
