Регламентированный учет

Используя данный раздел панели администрирования можно настроить параметры ведения регламентированного учета.

Отражение операций

Регламентированный учет - включает ведение регламентированного учета в системе.
Управляет использованием следующей функциональности:

  • Отражение документов в регл. учете
  • Закрытие месяца по регл. учету
  • Стандартные бухгалтерские отчеты
  • Регистры налогового учета
  • 1С-Отчетность
  • Специальные режимы налогообложения
  • Торговый сбор
  • Контролируемые сделки
  • Получение фактических данных бюджетирования по данным регл. учета.

В регламентированном учете могут отражаться как все хозяйственные операции, так и только операции, начиная с определенной даты. С помощью команды Ввести остатки по данным оперативного учета на дату начала ведения учета остатки по счетам регламентированного учета могут быть частино сформированы по данным оперативнового учета. При этом будут созданы документы Ввод остатков по регламентированному учету, заполненные остатками по соотвествующим регистрам оперативного учета.

Изменение использования регламентированного учета и даты начала ведения учета не рекомендуется после начала отражения хозяйственных операций в учете. При попытке изменения настроек проверяется наличие зарегистрированных к отражению документов в регл. учете. Если такие документы обнаружены, то при попытке изменения настроек выдается предупреждение.

При включении опции Управленческий учет на плане счетов регламентированного учета
на едином плане счетов регламентированного учета дополнительно ведется отражение операций в оценке управленческого учета.
Доступно отражение операций в оценке управленческого учета с определенной даты. С помощью команды Сформировать остатки на дату начала ведения учета остатки по счетам учета могут быть частично сформированы по данным оперативного учета.
Следует учитывать, что сформированные остатки требуют выверки. При необходимости остатки необходимо дополнить или откорректировать.

При включении опции Учет в валюте управленческой финансовой отчетности отражение операций на плане счетов выполняется с пересчетом по курсу в валюту управленческой финансовой отчетности. В поле Источник суммы для пересчета может быть выбран вид учета, данные которого пересчитываются в валюту управленческой финансовой отчетности.

Настройка счетов учета производства по группам продукции - включает дополнительную детализацию настроек счетов учета производственных затрат по группам аналитического учета номенклатуры.

Отражать в учете только проверенные документы - при групповом отражении документов в регламентированном учете обрабатывать только документы со статусом Проверен. При отражении документа из формы просмотра проводок статус проверки не анализируется.

Подробное описание приведено в статье Отметка о проверке документов бухгалтером https://its.1c.ru/db/erpdoc#bookmark:Bookkeeping:AccountingRecordsVerification

Отчетность

Формат сохранения регистров бухгалтерского учета - настройка формата сохранения электронных копий регистров бухгалтерского учета.

Контролируемые сделки - настройка правил для формирования списка контролируемых сделок, подготовка уведомлений для ФНС.

Подробное описание приведено в статье Контролируемые сделки https://its.1c.ru/db/erpdoc#bookmark:ControlledTransactions:ControlledTransactions

Торговый сбор - настройка включает функциональность учета торгового сбора в системе.

Подробное описание приведено в статье Торговый сбор https://its.1c.ru/db/erpdoc#bookmark:Taxes:SalesTax

Не использовать проверку выполнения операций закрытия месяца в отчетах - настройка отключает в бухгалтерских и налоговых отчетах вывод информации об актуальности выполнения регламентных операций закрытия месяца.

Сервисные функции

Ручное изменение проводок документов - настройка режима, при котором пользователи, обладающие достаточными правами, получают возможность изменять произвольным образом отражение документа в бухгалтерском и налоговом учете. При этом данные изменения фиксируются и дальнейшая работа с этим документом (или любых других операций влияющих на проводки при стандартном поведении системы) не приводит к формированию новых или изменению текущих проводок документа. Дальнейшее изменение проводок документа будет возможно только в ручном режиме, поэтому использование данной опции потребует повышенного внимания пользователей к отслеживанию изменения состояния системы влияющих на проводки документа и актуализации текущих проводок.

Подробное описание приведено в статье Ручная корректировка проводок в документах оперативного учета https://its.1c.ru/db/erpdoc#bookmark:Bookkeeping:ManualAccountingRecordsCorrection

УСН

Признание доходов отчетом о розничных продажах с - дата, начиная с которой доходы в книгу учета доходов и расходов будут попадать на основании отчета о розничных продажах (ранее использовался приходный кассовый ордер с хоз. операцией "Поступление из кассы ККМ"). Определяет движения документа в регистре "Книга учета доходов и расходов", а так же отображение в отчете "Книга учета доходов и расходов".

Подробное описание приведено в статье Учет по упрощенной системе налогообложения (УСН) https://its.1c.ru/db/erpdoc#bookmark:SimplifiedTaxSystem:SimplifiedTaxSystem
