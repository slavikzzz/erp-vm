Выполнение этапов производства

Рабочее место предназначено для отражения списка этапов производства, подлежащих выполнению, и осуществления действий по по их выполнению.

В рабочее место попадают этапы со статусами К выполнению, Начат и Завершен. Этапы в статусах Формируются и Сформирован не попадают в это рабочее место ни с какими отборами.

Действия

Отборы

Может быть установлен отбор по подразделению, исполняющему этап, состоянию этапа, признаку обеспеченности материалами и периоду выполнения (при включенном планировании графика производства)

Состояние выполнения

Состояние выполнения этапа определяется из статуса этапа и статусов предшествующих этапов. Состояние этапа может иметь следующие значения:

  • Ожидание предшествующих;
  • Начаты предшествующие;
  • Можно выполнять;
  • Начат;
  • Завершен.

Ожидание предшествующих

Такое состояние означает, что у этапа есть предшествующие этапы, и они еще даже не начаты. Это самое “дальнее” от возможного выполнения состояние.

Начаты предшествующие

Означает, что этапы, предшествующие выбранному, уже выполняются. Это уже на шаг ближе к возможному выполнению, так как когда завершатся предшественники, можно будет начинать выбранный этап.

В списке для этапов в состояниях Ожидание предшествующих и Начаты предшествующие используется серый шрифт, символизируя тот факт, что эти этапы пока рано выполнять, надо дождаться предшественников.

Можно выполнять

Означает, что у этапа нет незавершенных предшественников. Такой этап можно уже выполнять. Начиная с этого состояния для этапов в списке используется черный шрифт, так как над этими этапами уже можно работать.

Начат, Завершен

Состояния соответствуют одноименным статусам этапа Начат и Завершен.

Поля списка
  • Материалы не обеспечены - индикатор наличия в этапе материалов, которые все еще не обеспечены. При необособленном обеспечении это такие материалы, у которых в этапе стоит действие К обеспечению. При обособленном обеспечении те материалы, по которым все еще не созданы обеспечивающие заказы. Более подробную информацию об обеспечении этапа можно получить, открыв форму самого этапа, либо воспользовавшись командой Состояние обеспечения
  • Требуется оформить выработку сотрудников - индикатором обозначаются этапы, в трудозатратах которых отмечено выполнение видов работ сотрудников, но не оформлена выработка сотрудников.
  • Требуется рассчитать график производства - индикация этапов, требующих расчета или пересчета графика производства. Этот индикатор не подразумевает никаких действий в рабочем месте, но служит сигналом, что запланированные даты выполнения неактуальны.
  • Состояние производсвтенных операций - для этапов, выполняющихся в подразделениях с использование пооперационного управления, содержит индикацию о состоянии выполнения операций этапа.
  • - индикация появляется у этапов в статусе Начат, по коотрым еще не началось выполнение операций.
  • - по этапу начато выполнение операций.
  • - завершено выполнение всех операций в этапе. Такой этап можно закрывать.
  • Этап - обозначение этапа. Формируется из номера и названия.
  • Состояние - состояние выполнения этапа.

Совет. Рекомендуется использовать состояние выполнения для сортировки списка этапов. Это позволит наглядно поделить список этапов на несколько пулов:

  • которым еще долго до выполнения;

  • которые скоро можно будет выполнять;

  • к выполнению которых уже можно приступить;

  • выполняющиеся;

  • завершенные.


Такая сортировка в сочетании с отборами по состоянию выполнения позволяет акцентировать внимание пользователя на том, что важно сейчас, не отвлекаясь на весь массив информации.
  • Начало, Окончание- даты-время начала и окончания этапа, запланированная в графике производства. Колонки не выводятся при отключенном графике производства. Могут быть пустыми, если график производства еще не рассчитан.
  • Начало следующего этапа - запланированная в графике дата начала следующего этапа.

Информация о запланированном начале следующего этапа может использоваться для локального принятия решения об очередности выполнения этапов. Например, есть к выполнению 2 равных этапа. У первого этапа следующий этап запланирован на завтра, у второго этапа следующий только через месяц. При прочих равных предпочтительнее в первую очередь выполнить первый этап, чтобы уменьшить риск срыва графика.
  • Изделий - количество единиц/партий изделий, для которого создан этап. Это поле позволяет не заходя в сам этап быстро оценить объем работ по этапу.

Действия
  • Отметить начало выполнения - выбранные этапы переводятся в статус Начат. Если по каким-либо причинам действие не удается выполнить, будет выведено диагностическое сообщение.
  • Отметить выполненным - выбранные этапы отмечаются выполненным. При этом если в этапах не вносились вручную никакие изменения, система будет считать их выполненными без отклонений по материалам, выходным изделиям и трудозатратам, заполнив фактические величины из запланированных.
  • Оформить выработку сотрудников - при наличии выполненных видов работ сотрудников откроется рабочее место оформления выработки сотрудников для выбранных этапов.
  • Состояние обеспечения - открывается рабочее место Состояние обеспечения, с помощью которого можно управлять обеспечением выбранных этапов
