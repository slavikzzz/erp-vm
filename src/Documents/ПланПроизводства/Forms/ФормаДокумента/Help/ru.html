План производства

Документ предназначен для регистрации прогнозных показателей по производству продукции (полуфабрикатов), осуществлению ремонтов и разборок в фиксированном периоде.
Планы производства могут использоваться при "толкающей" схеме управления производством, когда потребность к производству создается внутри предприятия, заблаговременно до того, как возникает внешняя, "тянущая" потребность.

План производства формируется на период, в рамках которого планируемое количество детализируется до необходимой периодичности плана (недели, декады, месяца и т.д.). Например, если указанный период составляет один год, а периодичность месяц, то в каждой строке плана нужно будет ввести 12 помесячных планов.

Доступность документа определяется функциональной опцией Планы производства в разделе НСИ и администрирование - Бюджетирование и планирование - Планирование.

Использование Планов производства включает в остальных типах планов доступность операндов Планы производства (продукция), Планы производства (материалы), в формулах.

Функциональные особенности работы с документом на закладке Продукция аналогичны таковым для документа План продаж.

Заполнение табличной части по формулам


Заполнение плана может производится вручную или по формуле по кнопке Заполнить продукцию.
В виде плана необходимо указать простой вариант заполнения (по формуле).
По кнопке Заполнить продукцию можно заполнить состав номенклатуры:

  • по производимой продукции - позволяет заполнить номенклатуру, которая когда-либо производилась;
  • по признаку наличия спецификации - номенклатура, которая имеет хотя бы одну спецификацию;
  • из формулы - номенклатура будет получена из операндов (источников) указанных в формуле;
  • по отбору - номенклатура будет заполнена по произвольному отбору.
  • из плана/не менять - состав номенклатуры из плана не меняется.


Количество рассчитывается по указанной формуле.
Фактические данные выбираются из базы со смещением относительно периода планирования.

Заполнение табличной части по правилу заполнения (по источникам)


Для автоматического заполнения документа используется команда Заполнить продукцию - Настроить правило и заполнить.... Автоматическое заполнение производится на основании данных различных источников и правилах их преобразования.

В виде плана можно установить Запретить редактирование правила и кнопка Настроить правило и заполнить... будет не доступна.

Список полей, выбираемых различными источниками данных планирования, которые используются в правиле заполнения документа плана, должны совпадать с полями, загружаемыми в табличную часть документа.

Список полей, которые можно загрузить в табличную часть документа:

  • Период, если поле присутствует в выборке данных источника, то его значения должны попадать в интервал планирования, если поле не выбирается в источнике, то количество будет равномерно распределено между периодами планирования для периода в колонках;
  • Номенклатура, поле должно всегда присутствовать в выборке источника данных планирования;
  • Характеристика, если включена соответствующая функциональная опция;
  • Назначение, если в виде плана установлена детализация Объединенный план по назначениям;
  • Спецификация, если в сценарии планирования включен Расчет потребления материалов для планов производства;
  • Количество, поле должно всегда присутствовать в выборке источника данных планирования.

Разрезы планирования


При заполнении документа необходимо указать Сценарий планирования и Вид плана, в соответствии с которыми осуществляется заполнение остальных полей документа.

Заполнение плана может производится непосредственно по источникам данных (по кнопке Настроить правило и заполнить..), по формуле (по кнопке Заполнить продукцию) или вручную.
В виде плана для расширенного варианта можно настроить правило по умолчанию и установить запрет редактирования правила в документе, что позволит строго следовать правилу заполнения, установленного ответственным.

Пример. При расчете планируемых запасов на месяц нужно учитывать максимальный объем продаж в течение квартала, исключая при этом текущие складские остатки товаров и планируемые поставки товаров в течение периода планирования потребностей.
Для этого, в группу Максимум включаются 3 одинаковых источника с продажами за период, в качестве периода указываются последовательно 3 месяца из анализируемого квартала. Вне группы Максимум нужно добавить 2 источника из шаблонов Свободные остатки и Заказы поставщикам (планируемое поступление) со знаком минус.

Установка переключателя периода Еще - Периоды в положения: Периоды в колонках или Периоды в строках влияет на способ ввода данных по периодам.

Пример. Формируется еженедельный план на месяц. Если установлен переключатель периода в положение в колонках, то по каждой номенклатурной позиции указывается планируемое количество и сумма в колонках с указанием периода. Если установлен переключатель периода в строках, то тогда в табличной части необходимо дублировать позиции номенклатуры и в каждой строке указывать планируемое количество и сумму на дату начала периода.

При установке переключателя Еще - Периоды - Периоды в строках у пользователя появляется дополнительная возможность установки для каждой строки плана произвольной даты в рамках заданного периода (например, если период в плане установлен на месяц, то для каждой строки может быть задана произвольная дата внутри данного периода).

Для каждой ячейки плана можно ввести комментарий, в котором можно указать произвольные сведения.

Выделенные строки документа или ячейки плана можно отменить с помощью команды Отменить строки плана / Снять пометку на панели инструментов и контекстного меню - значение будет исключено (зачеркнуто).

Все не значащие строки, количество которых в каждом периоде = 0, можно автоматически удалить из документа. Меню Еще - Удалить пустые строки.

Спецификация заполняется в документ в зависимости от настроек в виде плана:

  • На дату планирования - если в документе плана несколько периодов, для строки продукции будет подобрана спецификация действующая на начало первого периода.
  • На каждый период - в каждом периоде могут быть выбраны различные спецификации, действующие на начало каждого периода.

Планирование полуфабрикатов

Полуфабрикаты, описанные в спецификации как производимые в процессе, разузловываются безусловно, встраивая свои спецификации в структуру изделия.
Полуфабрикаты обеспечиваемые (не производимые в процессе) планируются в зависимости от настройки планирования полуфабрикатов в виде плана.

При автоматическом планировании полуфабрикаты планируются отложено после проведения плана, одновременно с формированием потребности в материалах. Такие полуфабрикаты разузловываются в автоматическом режиме вплоть до закупаемых материалов. Автоматически запланированные полуфабрикаты отображаются в табличной части на закладке Полуфабрикаты. В этой табличной части можно вносить ручные изменения в план производства полуфабрикатов. В частности, можно менять спецификацию, по которой должен производиться полуфабрикат, даты запуска и выпуска, количество. Для строк, имеющих ручные изменения, отображается соответствующий признак в виде значка с карандашом.

При автоматическом пересчете плана производства ручные корректировки плана выпуска полуфабрикатов могут быть сброшены, о чем появится информационное сообщение на закладке Полуфабрикаты.

Команда Показать вхождения позволяет отобразить полуфабрикаты верхнего уровня для выделенной строки таблицы.

Полуфабрикаты участвуют в формировании заказов на производство по плану.

При выбранном ручном планировании становится доступной команда Запланировать полуфабрикаты, по которой можно дополнить список продукции полуфабрикатами, пропорционально запланированному количеству продукции.
Полуфабрикат (не производимый в процессе) будет добавлен в документ, если в настройках обеспечения для него задано:

  • Параметры управления запасами: Для склада материала, с которого потребляется полуфабрикат необходимо установить способ обеспечения с типом обеспечения = Производство
  • Для планируемого полуфабриката должна быть назначена основная спецификация
  • Для способа обеспечения на вкладке Ограничение использования в поле Подразделение необходимо указать такое же подразделение-диспетчер, которое управляет выпуском и на которое составлен план производства продукции


Команда Удалить полуфабрикаты позволяет удалить полуфабрикаты, добавленные при помощи команды Запланировать полуфабрикаты.


Команда Пересчитать потребность в материалах, видах РЦ и трудовых ресурсах позволяет пересчитать потребности при изменении НСИ.

Заполнение из внешнего табличного документа, буфера обмена, MS Excel


Команда Загрузить продукцию - Загрузить из внешнего файла - позволяет заполнить список продукции и другие реквизиты из любого табличного документа, вставив данные с помощью буфера обмена.
Команда Выгрузить в Excel и Загрузить из Excel позволяют сформированный пред заполненный план выгрузить в документ MS Excel через присоединенный файл, изменить по желанию, и загрузить в табличную часть документа. Если в документ MS Excel были добавлены новые строки - они будут распознаны автоматически. Для не сопоставленных значений необходимо указать соответствующие элементы НСИ вручную.

Заполнение по дефициту

Дефицитный план создается на один период. Для отображения доступен режим в строках.
Команда заполнения плана по дефициту перезаполняет план по имеющимся потребностям.

При редактировании дефицитного плана открывается специальная форма, где отображается текущее значение потребности и предлагается ввести скорректированное количество к обеспечению. Также вводится значения потребностей при достижении которых будет пересчитано заданное вручную количество. При изменении потребности в допустимом интервале пересчет не будет производится и план не будет сигнализировать о необходимости пересчета в рабочем месте и документе.

Если по плану в периоде есть незакрытый дефицит, то появляется информационное сообщение о необходимости пересчета плана.

Замещение планов


Если в плане выбран вид плана Замещающий, то планы будут заменять предыдущие в одинаковых периодах в разрезе измерений шапки плана в приоритете статуса и даты. Для замещенных периодов устанавливается признак Замещен. Перейти к замещенным и замещающим планам можно через гиперссылку Замещенные, Замещающие

Отражение потребности материалов


План производства при проведении может формировать потребность в материалах, видах рабочих центров и трудовых ресурсах. Для этого в сценарии, указанном в документе, должен быть установлен флаг Расчет потребностей в материалах, видах РЦ и трудовых ресурсах.

Поддерживается 3 стратегии определения потребности в материалах для плана производства: Наиболее вероятная, Оптимистичная и Пессимистичная.

  • Наиболее вероятная - в потребность попадают основные и альтернативные материалы, поправка на вероятность применятеся ко всем материалам.
  • Оптимистичная - планируются только основные материалы без поправки на вероятность. Альтернативные материалы не планируются
  • Пессимистичная. - Основные материалы планируются без поправки на вероятность. Альтернативные материалы планируются с поправкой на вероятность.

Создание заказов на производство по планам


По данным планов производства при помощи обработки Формирование заказов на производство по планам производится формирование заказов на производство.

Отчеты и печатные формы


Для документа План производства можно вывести печатную форму со списком товаров и периодами выпуска.
Исполнение плана производства можно отслеживать в отчете Исполнение плана производства с детализацией по подразделениям.

Если в сценарии, указанном в документе, установлен флаг Расчет потребностей в материалах, видах РЦ и трудовых ресусах, доступно формирование следующих контекстных отчетов
Потребность в видах РЦ
Потребность в материалах
Потребность в трудовых ресурсах
Потребность в полуфабрикатах


Отражение в бюджетировании


План производства при проведении может формировать плановые обороты по статьям бюджетов. Для этого в виде плана, указанном в документе, должен быть установлен флаг Отражать план в бюджетировании.

Корректировки потребностей

Помимо непосредственного редактирования количества к обеспечению в плане, можно скорректировать потребности. Ввод корректировки потребностей возможен командой в табличной части или из меню Создать на основании.
После ввода корректировки плановых потребностей табличная часть плана будет перезаполнена.
При изменении потребности в заменяемом товаре корректировка плановых потребностей переводится в статус К проверке, об этом сигнализирует информационное сообщение в документе.
