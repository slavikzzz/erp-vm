Заказ на производство (2.1)

Документ предназначен для регистрации потребностей по производству продукции/работ. Для исполнения заказов на производство строится график производства по данным заказов и информации о доступности ресурсов.
Заказы на производство могут быть внесены вручную, сформированы автоматически из обработки Формирование заказов по потребностям на основании анализа данных о потребности в продукции, созданы по планам производства, либо введены на основании других заказов: заказ клиента или заказ на ремонт.

В заказе на производство указывается подразделение-диспетчер, ответственное за производство.

Продукция


В табличной части Продукция указывается перечень изделий, которые необходимо произвести по этому заказу. По каждой строчке продукции указывается ресурсная спецификация, по которым необходимо произвести, количество, срок производства, склад для продукции, подразделение-получатель для работ, размещение выпуска.

Спецификация


По умолчанию в строке заказа указывается основная ресурсная спецификация, но может быть выбрана любая другая подходящая для выбранной номенклатуры действующая спецификация.
Спецификация заказа хранится в самом заказе и заполняется из указанной в строчке продукции ресурсной спецификации. Содержимое спецификации заказа может быть изменено без изменения ресурсной спецификации. По команде Изменить строку заказа открывается форма редактирования спецификации строки заказа, в которой можно внести изменения.
При наличии в спецификации заказа изменений относительно ресурсной спецификации, в колонке индикатора ручных изменений появляется соответствующий значок.
Строка заказа на производство с измененной спецификацией отражается зеленым цветом, если производство по ней еще не началось и редактирование спецификации строки может быть продолжено. После начала производства по строке ее цвет изменяется на черный.

При изменении количества продукции в строчке спецификация заказа перезаполняется из указанной ресурсной спецификации. При этом теряются внесенные вручную в спецификацию заказа изменения.

Если в заказе используются недействующие спецификации, то в форме заказа отображается индикатор Применяются недействующие спецификации, нажав на который можно открыть форму для замены недействующих спецификаций.

Срок производства


Срок производства определяется двумя датами: Начать не ранее и Дата потребности. При построении графика производства по данной строчке продукции этапы графика будут размещаться между указанными датами (с учетом буферов).

Размещение выпуска


Размещение выпуска задается для каждой строки продукции и может иметь значения К окончанию или К началу.

К началу означает, что этапы графика будут запланированы как можно ранее, начиная с даты Начать не ранее. При этом продукция может оказаться произведенной существенно раньше даты потребности.

К окончанию означает, что этапы графика будут планироваться так, чтобы продукция была готова как можно позже до даты потребности, и при этом все этапы по данной строке тоже планируются с как можно более поздним исполнением.
Но если при таком планировании "назад" для выпуска продукции к дате потребности потребуется начать производство ранее даты Начать не ранее, то строка будет перепланирована "вперед" от даты Начать не ранее.

Значение по умолчанию К началу

В одном заказе для разных строчек продукции может быть задано разное размещение выпуска.

Склад и подразделение -получатель


Для производимой продукции указывается склад, на который необходимо выпустить готовую продукцию. Для производимых работ необходимо указать подразделение-получатель.

Материалы

Материалы, необходимые для производства продукции, описываются в спецификации строки заказа. Заказ на производство в статусе "Создан" уже может обеспечитваться материалами и является распоряжением на отгрузку материалов со склада.
Управление передачей материалов в производство осуществляется с помощью рабочего места Получение и возврат материалов.
Информация о требуемых материалах может быть как измененая непостредственно в заказе, так и скорректирована с помощью корректировок заказа материалов.
Информация о датах обеспечения материалами из заказа (с учетом корректировок) используется при построении графика производства по заказу для определения даты, ранее которой не может быть запланирован этап.

Статусы заказа


Для Заказа на производство доступны статусы:

  • Создан - в этом статусе заказ представляет собой корректно заполненный документ, который пока еще не запускается в производство, по нему не строится график, не планируется поступление производимой продукции, но уже может обеспечиваться материалами.
  • К производству - по такому заказу строится график производства, резервируется доступность рабочих центров, в графике движения товаров появляется информация о планируемой доступности продукции.
  • Закрыт - заказ, по которому больше ничего не будет производиться. Статус "Закрыт" для заказа ставится вручную, служит для индикации того, что по заказу все сделано и для защиты заказа от того, чтобы по нему еще что-то делали. При проведении заказа на производство в статусе "Закрыт" выполняется контроль наличия невыполненных этапов графика производства и неоформленных распоряжений на выпуск продукции (при включенной функциональной опции "Не закрывать частично отработанные заказы на производство").

График производства


График производства строится по заказам в статусе "к производству". Представляет собой информацию о том, какие этапы когда исполнять, и какие ресурсы для этого требуются. График производства можно построить, проанализировать и диспетчировать из обработки Диспетчирование графика.

Управление приоритетами


Приоритет выполнения строится на основе очереди. Существует очередь исполнения заказов и очередь строк внутри заказа.

Приоритет не зависит от сроков производства, дат потребности. Заказ, который необходимо получить позже, может иметь более высокий приоритет, чем тот, который требуется раньше.

Для управления приоритетом исполнения строк внутри заказа необходимо расположить их в требуемом порядке в табличной части. Изменение приоритета всего заказа в общей очереди заказов делается в обработке Диспетчирование графика производства.

Смешивание приоритетов между заказами на уровне строк не поддерживается. Если необходимо одну часть исходного заказа сделать более приоритетной, чем другой заказ, а другую часть менее приоритетно, то исходный заказ надо разбить на два заказа, которые расположить в очереди в требуемом порядке.

Очередь исполнения заказов трехуровневая:

  • приоритет заказов,
  • очередь производственных подразделений,
  • очередь заказов.

Закрытие заказов


Установка для документа статуса Закрыт подразумевает перевод заказа в завершенное состояние, при котором все запланированные работы по нему выполнены или отменены, все распоряжения отработаны, а данные в информационной системе находятся в консистентном состоянии.

Для случая, когда часть работ по заказу еще не завершена, предлагается использовать механизмы:

  • сокращения производства;
  • ручной корректировки заказа на производства и распоряжений на передачу материалов;


В системе предусмотрен контроль закрытия заказов. Для использования контроля, необходимо предварительно включить функциональную опцию Не закрывать частично отработанные заказы на производство в разделе НСИ и администрирование - Производство. В этом случае заказу на производство невозможно будет назначить статус Закрыт, если по документу есть невыполненные этапы графика производства и неоформленные распоряжения на выпуск продукции.

Производство по заказу давальца


Заказ на производство может быть создан на основании Заказа давальца. В этом случае заказ давальца указывается в форме документа и в табличной части указывается продукция, заказанная давальцем и полуфабрикаты, которые производятся под эту продукцию. Вся продукция и полуфабрикаты, производимые по такому заказу, считается давальческой.

Давальческие материалы передаются в подразделение изготовитель обособленно под заказ давальца. В спецификации заказа на производство доступно редактирование перечня материалов и корректировка действий по обеспечению. В частности, можно определить, что часть материалов обеспечивается переработчиком. Такие материалы следует выделить отдельной строкой и назначить необходимое действие по обеспечению при помощи команды Заполнить обеспечение.
