Заказ материалов в производство

Документ предназначен для регистрации потребностей в материалах со стороны производства.

При включенной функциональной опции Управлением производством (версия 2.1) в разделе НСИ и администрирование – Производство потребность задается для производственного подразделения. Если включена опция Управлением производством (версия 2.2) в том же разделе, потребность задается для цеховой кладовой.


Исполнением заказа является документ Передача материалов производство при выбранной версии 2.1, либо в версии 2.2 документ Движение продукции и материалов с выбранной операцией Передача материалов в кладовую.

На основании заказа могут быть созданы заказы на передачу материалов в производство, если заказанный товар производится предприятием.

Статус К выполнению дает возможность начать отгрузку заказа для тех строк, у которых установлен вариант обеспечения (колонка Действия) Отгрузить или Отгрузить обособленно.
Если необходимо контролировать завершение работы по документу, возможно использование статуса Закрыт. Проведение документа в этом статусе доступно лишь в том случае, если количество запланированных к отгрузке товаров, совпадает с количеством отгруженных товаров (по документу Передача материалов в производство, а так же по документу Расходный ордер на товары (если для склада используется ордерная схема)).

Материалы на ИТС

Документ может быть создан в ручном (команда Создать-Новый) и автоматическом режимах (команда Создать - По плану).

В поле Склад указывается склад-отправитель материала, на котором создается потребность. Если используется производство версии 2.1, в поле Подразделение указывается подразделение-получатель, которому необходимы материалы. При использовании производства версии 2.2 указывается цеховая кладовая, в которую необходимо передать материалы.

Управление статусами заказа
  • К обеспечению - если по заказу существует потребность и ее необходимо обеспечить к определенной дате.
  • К выполнению - если потребность по заказу уже обеспечена (материал есть в наличии на складе) и его можно отгрузить к тем датам, которые указаны в табличной части документа.
  • Закрыт - если заказ отгружен.

Заполнение табличной части


Поле Назначение табличной части Материалы определяет назначение обособленного материала, списываемого со склада и передаваемого в производственное подразделение.

Возможны следующие варианты передачи обособленно материала:

  • Передача необособленного материала. При такой передаче поля Действие и Назначение в заказе и передаче не заполняются.
  • Обособление материала из свободного остатка на складе и обособление в производстве. Поле Действие в заказе устанавливается в не обособленно, а Назначение заполняется назначением, под которое материалы передаются в производство.
  • Обособление под заказ на передачу и снятие обособления при передаче. Поле Действие в заказе устанавливается в Обособленно, Назначение не заполняется.
  • Передача обособленного материала. Поле Действие в заказе устанавливается в Обособленно, Назначение заполняется назначением обособленного материала на складе. Под это назначение материал будет обособлен в производстве.


Новая номенклатурная позиция добавляется путем выбора из справочника Номенклатура (кнопка Добавить). Для добавления нескольких позиций в табличную часть документа можно также использовать диалоговое окно подбора (кнопка Заполнить - Подобрать товары).

  • Если для товара, указанного в заказе, ведется учет в разрезе характеристик, то дополнительно вводится информация о характеристике товара.
  • Для каждого товара указывается количество, которое заказывается. Из выпадающего списка выбирается упаковка, в которой заказывается товар. Если товар заказывается без упаковки, то из списка выбирается .
  • Для материалаов передаваемых в производство можно указать группу (вид) продукции для производства которой данные материалы передаются.


Контроль и корректировку графика отгрузки заказов можно осуществлять при помощи гиперссылки Состояние обеспечения. После выполнения рекомендаций результат переносится в редактируемый заказ по команде Перенести в документ.

Для управления обеспечением товарных позиций заказа предусмотрена колонка Действия. Для заказа материалов доступно указание следующих действий:

К обеспечению - позволяет зафиксировать потребность в товаре на складе-отправителе на дату, указанную в колонке Начало отгрузки,
Резервировать на складе - позволяет зарезервировать товар на складе-отправителе, для последуюшей передачи материалов по заказу,
Не обеспечивать - данное действие используется, если нет необходимости в фиксации потребности и предварительном резервировании товара на складе до его отгрузки.

При добавлении строк в список комплектующих автоматически устанавливается действие К обеспечению. Флаг Обособленно устанавливается если для номенклатуры установлена Обосоленная закупка/продажа.

Для документов созданных в рамках Управления производством версии 2.1. График отгрузки товаров задается непосредственно в таблице Материалы в колонке Дата отгрузки. При необходимости группового изменения значений сроков отгрузки, Даты отгрузки нескольких строк могут быть изменены с помощью команды Заполнить дату отгрузки группы команд Функции. Так же есть возможность в подвале таблицы Материалы указать общую дату, если требуется Отгружать одной датой.

В документах, создаваемых для целей Управления производством версии 2.2., возможно управление графиком отгрузки и поступления. Для этого в подвале таблицы Материалы требуется указать флаг использования произвольной Длительности перемещения (в днях). Значение поля Длительность используется для расчета даты Окончания поступления, при заполнении даты Начала отгрузки с помощью механизмов обеспечения: после подбора товаров, выбора действия обеспечения и выполнения команды Заполнить обеспечение. А так же для расчета даты Начала отгрузки от даты Окончания поступления командами Заполнить дату окончания поступления для выделенных и всех строк заказа. Длительность перемещения не обязательна к заполнению. В случае, когда её невозможно указать, по причине разных сроков поставки, Длительность можно оставить пустой.


Из группы команд Функции можно заполнить статьи расходов для всего заказа или только для выделенных строк.

Корректировка обеспечения

При необходимости произвести обособленное резервирование товара со свободных остатков склада или изъять резерв другого заказа в пользу текущего, либо снять резерв текущего заказа (в свободный остаток склада) можно воспользоваться следующими командами меню Обеспечение закладки Товары:

  • Снять резерв по назначению
  • Зарезервировать под назначение

Корректировка документа


В любой момент времени можно откорректировать количество в заказе и перепровести документ. При этом будут автоматически откорректированы данные по обеспечению или выполнению заказа. Для любой позиции в заказе можно установить признак Отменено. После проведения документа эта информация отобразится в графике движения товаров, менеджер по логистике будет иметь оперативную информацию об изменении заказа.

С помощью команды Отменить неотработанные строки можно выставить признак Отменено на те позиции, работа по которым не была произведена.

В ситуации, когда при вводе накладной (Передача материалов в производство) обнаружено отклонение количества от заказа (в пределах допустимых отклонений), для устранения этих расхождений можно скорректировать количество в табличной части заказа при помощи Помощника закрытия заказов (команда формы Закрыть заказ), которая получает информацию о фактической отработке по данными накладных и ордеров.
Корректировка при помощи обработки доступна, если задан процент допустимых отклонений в НСИ и администрирование - Номенклатура - Допустимое отклонение отгрузки и приемки мерных товаров. Обрабатываются только строки с мерными товарами.

Механизм контроля
В том случае, если не совпадает запланированное и отгруженное количество по товарным позициям, механизм контроля запрещает проведение.
Запланированное количество рассчитывается по товарным позициям заказа. Отгруженное количество по товарным позициям накладной. В том случае, если используется ордерная схема, дополнительно проверяется совпадение запланированного количества с количеством по ордеру.
