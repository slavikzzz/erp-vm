Заказ переработчику (2.4)

Документ предназначен для регистрации заказов в стороннюю организацию (именуемую Переработчиком), на оказание услуг по переработке переданного ему сырья и материалов, и выпуске готовой продукции (полуфабрикатов) и работ.
Права на сырье и материалы, переданные в производство на стороне, на всех этапах переработки, остаются за нашей организацией.
Заказ переработчику может быть создан на основании Заказов на производство, выступая в роли отдельного этапа собственного производства (если используется Управление производством 2.1.) и на основании Этапов производства (если используется Управление производством 2.2.), выступая частью одного производственного процесса.

Отражение получения услуг по переработке сырья и материалов производится в документе, если в реквизите Услуги по переработке выбрано значение "Указываются в заказе". При значении "Оформляются отдельно", скрывается закладка "Услуги" и элементы формы связанные с оплатой.

Документ не формирует проводок в регламентированном учете.

Закладка Основное


Поле Переработчик может быть заполнено только тем партнером, который является Поставщиком, поскольку он будет оказывать услугу по переработке.
Заказ переработчику может формироваться с указанием договора, который будет определять порядок расчетов.

Для возможности поступления продукции на разные склады в рамках одного заказа, необходимо включить функциональную опцию Поступление товаров на несколько складов в разделе НСИ и администрирование - Закупки - Документы закупок.

Если заказ содержит различную продукцию, то может возникнуть необходимость в группировке затрат на эту продукцию:

  • Без группировки - затраты будут распределены на всю продукцию заказа;
  • По продукции - затраты будут распределены на продукцию, к которой они относятся с учетом Назначения;
  • По спецификациям продукции - затраты будут распределены на продукцию, которая относится к указанной спецификации. Группировка доступна, если в качестве Варианта обособления сырья и материалов в переработке на стороне не выбрано значение По продукции;
  • По заказам на производство - затраты будут распределены на продукцию, которая относится к указанному этапу заказа. Группировка доступна, если используется Управление производством 2.1.
  • По этапам производства - затраты будут распределены на продукцию, которая производится на последнем этапе цепочки к которой относится выбранный этап. Группировка доступна, если используется Управление производством 2.2.

Закладка Выходные изделия

Если заказ находится в статусе К исполнению или выше, становится обязательным поле Дата поступления. Его можно указать как для всего документа в целом, так и для каждой отдельной строки. При выбранной группировке затрат По продукции в состав ключевых полей помимо Номенклатуры и Характеристики включено Назначение продукции, при этом одинаковые строки продукции, разделенные, например, из-за разных дат поступления, принадлежат одной группировке.

Номенклатурные позиции с типом номенклатуры Работа могут быть списаны на расходы. Для этого необходимо установить соответствующий флаг и выбрать статью расходов с аналитикой расходов

Для каждой строки табличных частей заполняется номенклатурная позиция, которая будет произведена из переданного сырья и материалов. Для удобства заполнения табличной части, можно использовать Подбор товаров.

Список Побочный и промежуточный выпуск по фиксированной стоимости предназначен для дополнительных выходных изделий, стоимость которых фиксирована и определяется в момент выпуска продукции. Такие изделия уменьшают себестоимость продукции на указанную стоимость.

При оформлении заказа переработчику без привязки к документам Этап производства для возвратных отходов может быть указана цена и сумма, которая будет использована для расчета учетной стоимости возвратных отходов при оформлении документа Поступление от переработчика на основании заказа переработчику. Статья калькуляии обязательна к заполнению в случае если в настройках программы указано использование Партионного учета версии 2.2.

Так же в табличной части возвратных отходов указывается Статья калькуляции для последующего заполнения статьи калькуляции в отчете переработчика. При вводе на основании заказа на производство статья калькуляции автоматически заполняется по данным спецификации заказа.

Закладка Сырье и материалы


В табличной части указывается сырье, которое наша организация готова предоставить переработчику. Также может быть заполнена Залоговая стоимость передаваемого сырья. Склад отгрузки указывается отдельно в каждой строке (для удобства заполнения, можно выделить несколько строк, использовать команду Заполнить - Склад, выбрать склад и нажать Выбрать).

Так же в табличной части указывается Статья калькуляции для последующего заполнения статьи калькуляции в Отчете переработчика. При вводе на основании Заказа на производство Статья калькуляции автоматически заполняется по данным спецификации заказа. Статья калькуляии обязательна к заполнению в случае если в настройках программы указано использование Партионного учета версии 2.2.

Если использовать собственное сырье не планируется и производство продукции на стороне планируется полностью из сырья и материалов переработчика, то в этом случае следует использовать Заказ поставщику.

Для строк с указанным действиями обособленного обеспечения или отгрузки, Назначение выбирается в соответствии с заданным в настройках вариантом обособления сырья в переработке на стороне. Для варианта обособления по переработчику/договору в качестве Назначения будет выбран указанный Договор, если он не заполнен – Партнер. Для варианта по назначению продукции будет использоваться целевое назначение продкуции, а если в соответствующей обособляемому материалу группе затрат не указано назначение, обособление будет выполнено под текущий заказ переработчика.

Закладка Услуги по переработке


На закладке указываются наименования услуг по переработке, общая Сумма, Ставка НДС и Сумма НДС услуги, а так же Статья калькуляции, по которой услуга включается в себестоимость полученной от переработчика продукции.

Закладка Дополнительно


На закладке указывается Выпуск под деятельность, Валюта и Подразделение, в рамках которого будут собираться все затраты по переработке переданного сырья. Поля Желаемая дата отгрузки и Желаемая дата поступления могут быть заполнены и, в этом случае, будут учтены при формировании плановых дат отгрузки и поступления.

В Подразделении, которое указывается в заказе переработчику, будут собираться все затраты в виде стоимости переданного сырья и далее будут распределены на поступившую продукцию. Подразделение в данном случае выступает в качестве инициатора переработки.

Если документ вводится на основании графика производства, то в поле Запланировано можно отрегулировать количество партий/единиц продукции, которые будут обеспечены этим заказом. Перед установкой статуса заказа Закрыт необходимо будет указать в поле Выполнено фактически произведенное на стороне количество партий/единиц продукции.

Корректировка заказа

В процессе исполнения заказа можно изменять действия по обеспечению и отгрузке заказа непосредственно в строке списка сырья и материалов, либо перезаполняя обеспечение. Также можно отменить исполнение строки заказа, установив флаг Отменено. При проведении скорректированного документа будет обновлена информация о необеспеченных потребностях, график отгрузки сырья и материалов и график поступления выходных изделий по заказу.

Для автоматической отмены всех недопоставленных выходных изделий необходимо использовать команду Отменить непоставленные строки. Аналогично, для автоматической отмены всех неотгруженных сырья и материалов необходимо использовать команду Отменить неотработанные строки. Данные команды удобно использовать для закрытия заказа перед установкой статуса Закрыт. В случае если установлен запрет на закрытие частично отработанных заказов (функциональная опция Не закрывать частично отработанные заказы переработчикам в разделе НСИ и администрирование - Производство - Переработка на стороне), при проведении заказа переработчику в статусе Закрыт осуществляется контроль отгрузки и приемки по заказу: строки выходных изделий заказа должны быть приняты, либо отменены, а строки сырья и материалов должны быть отгружены либо отменены.

Статусы и приоритеты

Для указания состояния документа предусмотрены следующие статусы:

  • на согласовании - заказ сформирован, но еще не согласован с переработчиком;
  • к исполнению - ожидается передача сырья и материалов переработчику для начала производства. Параллельно считается, что наша организация готова принимать выпущенную продукцию от переработчика. В этом статусе информация о потребностях попадает в рабочее место по оформлению Передач сырья переработчику и в рабочее место по оформлению Поступлений от переработчика;
  • закрыт - сформирован Отчет переработчика.


Доступность статусов определяется функциональной опцией Статусы заказов переработчикам в разделе НСИ и администрирование - Производство - Переработка на стороне.

При работе с заказами можно установить приоритетность выполнения заказов. В дальнейшем пользователь может сортировать заказы в зависимости от их приоритета.
В процессе работы, менеджер может отменить отдельные позиции по заказу, После окончания работы по заказу пользователь должен установить для заказа статус Закрыт. Если заказ выполнен не полностью, то пользователь должен указать причину, по которой невозможно было обеспечить заказ полностью (причина отмены).

Передача материалов

Проведение заказа со статусом К исполнению является распоряжением к началу оформления складской службой документов по передаче сырья переработчику.

Список распоряжений, по которым можно отгружать сырье, отображается в рабочем месте Передачи сырья переработчикам на закладке К оформлению.
