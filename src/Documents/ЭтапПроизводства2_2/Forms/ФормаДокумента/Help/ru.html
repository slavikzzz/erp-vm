Этап производства

Этап производства является документом, предназначенным для управления ходом производства. Этап создается и выполняется в рамках заказа на производство. При использовании ресурсных спецификаций этап производства формируется при помощи информации из этапа спецификации.

Этап производства используется как для планирования производства в графике, так и для отражения фактического выполнения этапа. Для целей планирования графика этап содержит в себе описание параметров производственного процесса (длительность, ресурсы). График производства представляет собой расстановку этапов производства на оси времени с определением дат начала и завершения.

Этап содержит в себе информацию о материалах, которые требуется обеспечить для его исполнения, и является распоряжением на отгрузку этих материалов. Этап может обеспечиваться обособленно, при этом сам выступает назначением для обеспечения. В этапе описываются выходные изделия, которые должны возникнуть при его исполнении.
При выполнении в самом этапе производства указывается информация о расходовании материалов и о возникновении выходных изделий.
По этапу могут описываться трудозатраты для его выполнения: плановые и фактические.

Порядок работы с документом описан в документации на ИТС https://its.1c.ru/db/erpdoc#bookmark:Version22StagesOfProduction:Version22StagesOfProduction

Отдельные этапы могут выполняться от имени разных организаций по схеме внутренней переработки. Внутренняя переработка описана в документации на ИТС https://its.1c.ru/db/erpdoc#bookmark:Outsourcing:Outsourcing

Основное

Номер

Номер документа. При автоматическом формировании этапов номер генерируется из трех частей, разделенных точкой: номер заказа на производство, номер цепочки этапов внутри заказа, номер этапа в цепочке.
В списках этап представляется при помощи этого номера и названия.

Спецификация

Отображается гиперссылка на ресурсную спецификацию, при помощи которой был сформирован этап. Если этап создавался без применения спецификации, будет написано "нет спецификации". В этом же поле присутствует название основного выходного изделия цепочки этапов и гиперссылка “показать этапы”, по которой открывается форма Последовательность этапов, содержащая список этапов цепочки, к которой относится данный этап.

Маршрутная карта

Поле Маршрутная карта доступно, если в подразделении, исполняющем этап, включено использование пооперационного управления. В этом поле необходимо выбрать маршрутную карту, содержащую описание операций, при помощи которых выполняется этап.

В поле Коэффициент задается коэффициент приведения нормативов маршрутной карты к выполнению этапа. При формировании операций для этапа нормативы из маршрутной карты будут умножены на этот коэффициент.

Маршрутная карта описана на этап приготовления раствора с нормативами на 10 кг.
По этапу производства необходимо приготовить 500 кг раствора. В таком примере в этапе нужно указать коэффициент 50. При формировании операций для этого этапа все нормативы выполнения операций, включая расход материалов, трудозатраты, выход изделий и количество выполнения операций будет умножено на 50.

Планирование этапа

В поле Планировать выбирается вариант планирования этапа в графике производства.


  • По общей длительности - планирование этапа осуществляется по длительности, без учета доступности видов рабочих центров. Длительность может быть задана в днях, часах или минутах. Если длительность этапа задана в днях, то при планировании работы используется календарь подразделения, из которого определяются рабочие дни, без подсчета рабочего времени дня. Если длительность этапа задана в минутах или часах, то при планировании работы используется точное расписание работы подразделения, с указанием времени.

  • По доступности ВРЦ - планирование осуществляется с учетом доступности видов рабочих центров. При выборе этого варианта становится доступной закладка Виды рабочих центров, на которой задаются необходимые для такого планирования параметры.

  • Вручную - при выборе этого варианта осуществляется ручное размещение этапа в графике. Для этого по гиперссылке перейти к размещению в графике открывается соответствующая форма.


При автоматическом планировании этапа можно задать дату планировать не ранее. Это дает возможность учесть любые внесистемные факторы, из-за которых плановая дата запуска этапа должна быть подвинута на более поздний срок.

Информация о планируемом периоде производства по этапу отображается в поле Производство. Период отображаеься в виде гиперссылки, по которой открывается форма размещения этапа в графике.
Фактическое завершение работ по этапу отмечается в полях Начат и Завершен. Заполнение этих полей взаимосвязано со сменой статуса этапа.
В поле Причина задержки может быть выбрана причина, по которой выполнение этапа завершено не вовремя. Настройка списка возможных причин доступна в разделе Производство - Настройки и справочники - Причины задержки выполнения этапов производства.

Обеспечение материалами и работами

Закладка Обеспечение материалами и работами предназначена для указания перечня материалов, работ и полуфабрикатов, необходимых для выполнения этапа, и для управления их обеспечением. Список материалов, необходимых этапу, заполняется при формировании из ресурсной спецификации и может быть произвольно изменен вручную. Команда Заполнить по спецификации позволяет перезаполнить список материалов по данным ресурсной спецификации. С помощью команды Заполнить склад можно заполнить склад в выделенных строка.
Исходное количество материала из ресурсной спецификации можно увидеть в колонке Норматив, которая появляется в списке при нажатой кнопке Показать нормативы.

Если в ресурсной спецификации указана номенклатура с типом Набор, то при формировании этапа, состав набора переносится в документ в виде отдельных позиций.

Полуфабрикаты

В списке могут присутствовать как материалы, создающие штатно обеспечиваемую потребность на складе, так и материалы, являющиеся полуфабрикатами, производящимися в рамках одного заказа на производство с потребляющим их этапом. Для таких производимых полуфабрикатов в колонке Производится по спецификации или на этапе отмечается флажок Производится. При этом, если указана спецификация, по которой должен производиться полуфабрикат, для его производства формируются необходимые этапы согласно выбранной спецификации. Если спецификация для производимого полуфабриката не указана, в поле выводится надпись "Производится на этапе" и система будет считать, что этот полуфабрикат будет произведен на предыдущих этапах этой же цепочки этапов в качестве промежуточного выхода.

Аналоги материалов

В колонке, обозначенной значком , отображается индикатор для материалов, имеющих аналоги, описанные в разрешениях на замену. Подбор аналогов для материала осуществляется с помощью команды Заполнить - Подобрать аналоги.

Желаемая дата обеспечения

Поле Дата отгрузки содержит дату, к которой должны быть обеспечены материалы. При автоматическом формировании этапов эта дата заполняется исходя из длительностей этапов, их последовательности в производственном процессе и заданной в заказе на производство даты Начать не ранее и Желаемой даты выпуска. Дата отгрузки может быть изменена вручную. Эта дата учитывается при обеспечении потребности в материалах.
Если по этапу рассчитан график производства и начало этапа запланировано позже даты отгрузки, рядом с датой появляется гиперссылка, позволяющая установить ее равной дате начала выполнения этапа по графику. Это может потребоваться, чтобы избежать заблаговременного резервирования материалов под этап.

Управление обеспечением

Для управления обеспечением используется колонка Действие, в которой указывается вариант обеспечения. Для заполнения варианта обеспечения выбранных материалов необходимо выполнить команду Обеспечение - Заполнить обеспечение.
Если в строке установлено действие К обеспечению , то в колонке Доступно отображается свободный остаток с учетом ранее распределенного резерва.
Для контроля обеспечения этапа материалами используется команда Состояние обеспечения. В форме обеспечения можно посмотреть дату поступления, обеспечивающий заказ или установить Отгрузить / Не обеспечивать, в зависимости от необходимых действий с материалами.

После этого программа заполнит варианты обеспечения и даты отгрузки в строках, при необходимости разбив строки по вариантам обеспечения и датам отгрузки. Для недостающих материалов будет указан вариант обеспечения К обеспечению и рассчитана дата отгрузки в соответствии с плановой датой поставки товара по способу обеспечения. При заполнении даты отгрузки будет учтена Желаемая дата обеспечения заказа и необходимость Отгружать одной датой.


Отгрузка материалов в производство

Для получения со склада материалов, необходимых этапу производства, в колонке Действие устанавливается вариант Отгрузить. Если склад, указанный для материала, является обычным складом или же кладовой другого подразделения, то действие Отгрузить создает распоряжение к началу оформления складской службой документов по отгрузке материала со склада в подразделение. Установка действия Отгрузить для склада, являющегося цеховой кладовой подразделения, выполняющего этап, приводит к списанию указанного количества материалов с остатков кладовой без необходимости оформления каких-либо дополнительных документов.

Отгрузка материалов в подразделение для исполнения этапа осуществляется только для начатых этапов. Если этап еще не начат (статус Формируется, Сформирован или К выполнению), при установке действия Отгрузить статус автоматически будет переведен в Начат. Если статус этапа перевести обратно в статус ниже начатого, действия Отгрузить будут заменены на К обеспечению.

Для перевода этапа в статус Завершен всем материалам должно быть выставлено действие Отгрузить. Если обеспечиваемые материалы к завершению этапа не были отгружены в полном объеме, необходимо откорректировать список материалов в обеспечении, убрав или отменив те, которые не отгружались.

Расход материалов и работ

Закладка появляется у этапов в статусе Начат и предназначена для отражения информации о фактическом расходе и экономии материалов. На закладке находятся 2 списка: Израсходованные материалы и работы и Неизрасходованные материалы.

Израсходованные материалы и работы

В списке заполняются фактически израсходованные при выполнении этапа материалы и выполненные для этапа работы. Обязательно указание даты расходования. При снятом флажке Расход одной датой в списке появляется колонка Израсходован, в которой для каждой строки должна быть указана дата расходования. Если материал расходовался порциями в разные даты, его необходимо разбить на разные строки.
Расход материалов может заполняться вручную либо по команде Заполнить отгруженными заполняется материалами, которым на закладке обеспечения установлено действие Отгрузить. Расход материалов не должен превышать обеспечение. Контроль этого условия осуществляется при переводе этапа в статус Завершен. Если список расхода не заполнялся, то при переводе этапа в статус Завершен он будет автоматически заполнен обеспечиваемыми материалами.
Содержимое списка определяет прямые материальные затраты по этапу, учитываемые при расчете себестоимости.

Неизрасходованные материалы

Список предназначен для отражения материалов, которые передавались на исполнение этапа, но не были израсходованы. С помощью этого списка можно отразить экономию материала, недорасход по причине неполного выполнения этапа или остатки, возникающие по причине передачи заведомо превышающего количества материалов.
Для неизрасходованных материалов доступны действия: Передать на склад, Списать на статью. При списании на статью указывается статья расходов. Для передачи на склад необходимо указать склад. Если в качестве склада указана цеховая кладовая подразделения, исполняющего этап, израсходованный материал вернется на остатки кладовой без оформления дополнительных документов. Если указать обычный склад или кладовую другого подразделения, то возникает распоряжение на оформление документа возврата материалов из производства. Неизрасходованные материалы могут быть возвращены как в свободный остаток на складе, так и обособленно под назначение этапа. Для обособленного возврата необходимо установить флажок в колонке Назначение.
Список неизрасходованных материалов может заполняться вручную либо по команде Заполнить неизрасходованными будет заполнен разницей между обеспечением и расходом.

При завершении этапа все отгруженные материалы должны быть либо израсходованы, либо оформлены неизрасходованными. Проверяется условие Отгружено = Израсходовано + Сэкономлено. Для контроля используется контекстный отчет Контроль движения продукции и материалов.

Выходные изделия

Закладка описывает изделия, возникающие как результат выполнения этапа. Выходные изделия этапа делятся на 2 категории, представленные отдельными списками: Продукция по рассчитываемой стоимости и Побочный и промежуточный выпуск по фиксированной стоимости. При формировании этапа выходные изделия заполняются из соответствующих табличных частей ресурсной спецификации.

Продукция по рассчитываемой стоимости

Список содержит продукцию, являющуюся основным результатом выполнения цепочки этапов. Себестоимость такой продукции рассчитывается. Продукция по рассчитываемой стоимости может выходить только с одного, последнего этапа цепочки (выпускающего). Во всех остальных этапах такой список отсутствует. Продукция в этом списке может являться конечной продукцией заказа на производство, может быть полуфабрикатом, потребляемым в этапе другой цепочки этого же заказа.

Рассчитанная себестоимость партии производства распределяется между выходными изделиями этап пропорционально доле стоимости. Доля стоимости обязательна для заполнения, если в этапе более одного выходного изделия по рассчитываемой стоимость. Доля стоимости является безразмерной числовой величиной.

Побочный и промежуточный выпуск по фиксированной стоимости

Табличная часть предназначена для побочных и промежуточных выходных изделий, стоимость которых фиксирована и определяется в момент выпуска продукции. Такие изделия уменьшают себестоимость продукции на указанную стоимость. Цены за единицу изделия задаются в колонке Цена. Цены могут быть заполнены вручную или автоматически по виду цен. Если на закладке Дополнительно заполнен Вид цен, цена заполняется автоматически при установке отметки Произведено в строке, по команде Заполнить по виду цен, при заполнении этапа по выполненным операциям либо при переводе этапа в статус Завершен.
Выход по фиксированной стоимости может осуществляться с любого этапа, не только с последнего. Изделия промежуточного выпуска могут предназначаться для последующих этапов этой же цепочки.

Выходные изделия обоих списков могут выпускаться в свободный остаток.

Для отражения факта производства выходного изделия отмечается флажок произведено и указывается дата производства. Дата производства может быть задана одна на все изделия или задавать построчно в выходных изделиях, позволяя отразить выход разными порциями в разные даты. При отметке производства выходного изделия этап, который еще не начат, автоматически переводится в статус Начат.
В поле Направление выпуска для производимых работ может быть указано В подразделение или Списать на статью, для товаров указывается На склад или Списать на статью.
Если склад является цеховой кладовой подразделения-исполнителя этапа, при отметке производства выходные изделия образуются на остатках кладовой без необходимости оформления каких-либо дополнительных документов. Если же указан обычный склад или кладовая другого подразделения, при производстве выходного изделия образуется распоряжение на оформление передачи продукции из производства.
Для завершения этапа все выходные изделия отмечаются произведенными. То, что не произведено к завершению, необходимо удалить из выходных изделий.

Особенности распределения затрат на выпуски в разных месяцах

Если продукция по этапу производится в течение одного месяца, то ее себестоимость будет сформирована из фактических затрат. Если производство продукции по этапу происходит частями в несколько месяцев, то закрытие производится по плановым данным. Списание по плановым данным будет производиться таким образом, чтобы "выровнять" фактически понесенные затраты с плановыми данными.

Плановые цены*

Если в учетной политике организации установлена настройка учета готовой продукции по плановой стоимости, то в документе появляется возможность указывать плановые цены выпуска продукции по рассчитываемой стоимости:

  • вручную;
  • заполнять из вида цен документа;
  • заполнять по плановой калькуляции.

Корректировка документа

В ситуации, когда при вводе накладной (Передача продукции из производства, Передача материалов в производство, Возврат материалов из производства) обнаружено отклонение количества от этапа (в пределах допустимых отклонений), для устранения этих расхождений можно скорректировать количество в соответствующей табличной части этапа и перепровести документ.
Это можно сделать как вручную, так и при помощи Помощника закрытия заказов (команда формы Закрыть этап), которая получает информацию о фактической отработке по данными накладных и ордеров.
Корректировка при помощи обработки доступна, если задан процент допустимых отклонений в НСИ и администрирование - Номенклатура - Допустимое отклонение отгрузки и приемки мерных товаров. Обрабатываются только строки с мерными товарами.

Трудозатраты

На закладке перечисляются виды работ сотрудников, необходимые для выполнения этапа. Список заполняется при формировании этапа из ресурсной спецификации. Может быть произвольно отредактирован вручную, а также перезаполнен по команде Заполнить по спецификации.
Исходное количество трудозатрат из ресурсной спецификации можно увидеть в колонке Норматив, которая появляется в списке при нажатой кнопке Показать нормативы.
После начала выполнения этапа можно отмечать выполнение видов работ, установив флажок в поле Выполнено, дату выполнения и указав исполнителя в поле Исполнитель. Для выделенных строк доступно групповое заполнение исполнителей и статей калькуляции по командам Заполнить исполнителя и Заполнить статью калькуляции.

При включенной опции Оформлять выработку сотрудников на основании производственных операций (НСИ и администрирование - Производство - Операции) в документе Этап производства при использовании пооперационного управления остаются только плановые данные по трудозатратам, отключается учет фактических данных трудозатрат и контроль выполнения трудозатрат.

В зависимости от настроек производственного подразделения в качестве исполнителей видов работ могут быть выбраны бригады или работники организации.
Отметка выполнения вида работ в неначатом этапе автоматически переводит его в статус Начат. По выполненным видам работ этапа образуются распоряжения на оформление выработки сотрудников. Рабочее место по оформлению выработки сотрудников вызывается из раздела Производство - Трудозатраты командой Выработка сотрудников.

Виды рабочих центров

Закладка доступна при выбранном варианте планирования этапа по доступности видов РЦ. В списке заполняются виды рабочих центров, доступность которых надо занять при планировании графика производства, с указанием времени работы, занимаемого на этих видах РЦ. При планировании занятости видов РЦ в графике учитываются величины предварительного и завершающего буфера. После начала этапа откладывается время предварительного буфера, потом занятость всех видов РЦ, и в конце этапа откладывается завершающий буфер.

Допускается указание загрузки нескольких видов РЦ в одном этапе. При этом необходимо определить порядок загрузки. Предусмотрены следующие варианты:
Одновременно - подразумевается одновременная работа всех перечисленных видов РЦ. Задается общее время работы. Для размещения подбираются интервалы, в которых одновременно доступны все виды РЦ
Последовательно - виды РЦ загружаются в порядке следования в табличной части. Загрузка каждого следующего вида РЦ начинается с интервала в котором размещен предшественник, включая данный интервал
Независимо - Порядок загрузки видов РЦ не имеет значения. Загрузка размещается независимо друг от друга в первый доступный интервал (несколько интервалов).

Печатные формы

Лист сопровождения партии - предназначена для сопровождения и идентификации партии деталей в производстве. Может использоваться для поиска документа по штрихкоду.
