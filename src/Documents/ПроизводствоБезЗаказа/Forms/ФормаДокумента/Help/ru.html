Производство без заказа

Документ предназначен для отражения производства продукции или работ и списания материалов с баланса складов и цеховых кладовых, а также работ с баланса подразделений, и их отнесения на выпущенную продукцию.
Помимо материалов и работ в документе указываются выполненные работы сотрудников (трудозатраты), которые требуется включить в себестоимость продукции. Документ является распоряжением на оформление документа выработки сотрудников.
Все затраты, отраженные документом, включаются в себестоимость выпущенной продукции в месяце отражения (не поддерживаются производственные циклы, переходящие через границу месяца).

Порядок работы с документом описан в документации на ИТС https://its.1c.ru/db/erp25doc#bookmark:ProductionWithoutOrders:ProductionWithoutOrders

В случае производства продукции под назначение, связанное с заказом давальца, доступно оформление отчета давальцу.

Документ может использоваться для оформлении внутренней переработки из давальческих материалов другой собственной организации. Производство по давальческой схеме описано в документации на ИТС https://its.1c.ru/db/erpdoc#bookmark:Outsourcing:Outsourcing

Заполнение документа

Группировка затрат

Используется для обособления затрат внутри партии производства. Возможные варианты:

  • По спецификациям - возвратные отходы, материалы и работы, трудозатраты привязываются к спецификации продукции;
  • По спецификациям и назначениям - возвратные отходы, материалы и работы, трудозатраты привязываются к спецификации продукции и назначению;
  • По продукции - возвратные отходы, материалы и работы, трудозатраты привязываются к продукции;
  • По продукции и назначениям - возвратные отходы, материалы и работы, трудозатраты привязываются к продукции и назначению;
  • По документу - при использовании данной группировки затрат колонка Группа затрат скрывается, возвратные отходы, материалы и работы, трудозатраты распределяются пропорционально долям стоимости, указанным для продукции;
  • Без группировки - при использовании данной группировки затрат табличные части Побочный и промежуточный выпуск, Материалы и работы, Трудозатраты скрываются, документ оформляется для формирования партии производства для дальнейшего использования при оформлении документов Распределение материалов и работ, Распределение расходов, Распределение возвратных отходов.

Для привязки затрат к группе продукции используется форма выбора группы затрат.

Продукция

Списки продукции и возвратных отходов, на которую относятся затраты. В целом форма документа производства без заказа соответствует форме Ресурсной спецификации.
При ручном оформлении документа для продукции склад по умолчанию заполняется кладовой цеха.
Для работы с табличными частями предусмотрены следующие команды:

  • Команда Заполнить группу затрат - предназначена для выбора и заполнения группы затрат в выделенных строках.
  • Команда Заполнить цены по виду цен - предназначена для заполнения цен побочного и промежуточного выпуска согласно виду цен, указанному в документе. Цены по виду цен получаются на дату документа.

Материалы и работы

Материалы и работы, затраченные на производство продукции.
В поле Склад указывается склад, либо цеховая кладовая.
В поле Подразделение затрат для материалов указывается подразделение, которое выступает получателем затрат (для цеховых кладовых устанавливается равным подразделению, указанному в карточке цеховой кладовой). Для работ указывается исполнитель работ, затраты которого должны быть списаны на данную партию.


Стоимость материалов включается в себестоимость продукции пропорционально доли стоимости (для разных наименований продукции) и количеству. Для интерактивной работы со списком предусмотрены следующие элементы управления:

  • Команда Дополнить по спецификации - предназначена для получения нормативов расхода материалов по спецификациям, указанным в табличной части Продукция, их приведению к количеству продукции в документе, подбору обособленных под назначение продукции остатков нормативных материлов и помещения результата в табличную часть Материалов и работ.
  • Команда Дополнить обособленными остатками - предназначена для получения обособленных под выпущенную продукцию остатков материалов и помещения этих остатков в табличную часть Материалов и работ.
  • Команда Заполнить группу затрат - предназначена для выбора и заполнения группы затрат в выделенных строках.
  • Команда Заполнить статью калькуляции - предназначена для выбора и заполнения статьи калькуляции в выделенных строках.

Трудозатраты

Список выполненных работ сотрудников по производству продукции. Окончательное включение трудозатрат в себестоимость продукции выполняется при расчете себестоимости. В зависимости от настроек производственного подразделения в качестве исполнителей видов работ могут быть выбраны бригады или работники организации.

  • Команда Дополнить по спецификации - предназначена для получения нормативов трудозатрат по спецификациям, указанным в табличной части Продукция, их приведению к количеству продукции в документе и помещения результата приведения в табличную часть Трудозатраты.
  • Команда Заполнить группу затрат - предназначена для выбора и заполнения группы затрат в выделенных строках.
  • Команда Заполнить статью калькуляции - предназначена для выбора и заполнения статьи калькуляции в выделенных строках.

В заголовке закладок по мере заполнения документа показывается информация о количестве строк табличных частей.

Дополнительно

Направление деятельности - при группировке затрат "По спецификациям", "По продукции", "По документу", "Без группировки" используется для учета затрат списанных на партию производства, но еще не отнесенных на конкретную продукцию.
Вид цены - используется для заполнения цен в табличной части "Побочный и промежуточный выпуск по фиксированной стоимости"

Плановые цены*

Если в учетной политике организации установлена настройка учета готовой продукции по плановой стоимости, то в документе появляется возможность указывать плановые цены выпуска продукции по рассчитываемой стоимости:

  • вручную;
  • заполнять из вида цен документа;
  • заполнять по плановой калькуляции.

Примечание. * данное поведение актуально для версии 2.5.х
