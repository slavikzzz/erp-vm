Заказ переработчику

Документ предназначен для регистрации заказов в стороннюю организацию (именуемую Переработчиком), на оказание услуг по переработке переданного ему сырья и материалов, и выпуске готовой продукции (полуфабрикатов) и работ.
Права на сырье и материалы, переданные в производство на стороне, на всех этапах переработки, остаются за нашей организацией.

Порядок работы с документом описан в документации на ИТС https://its.1c.ru/db/erpdoc#bookmark:OutsourcedProduction:OutsourcedProduction25

Заказ переработчику может быть создан на основании Этапов производства, выступая частью одного производственного процесса.

Отражение получения услуг по переработке сырья и материалов производится в документе, если в реквизите Услуги по переработке выбрано значение "Указываются в заказе". При значении "Оформляются отдельно", скрывается закладка "Услуги" и элементы формы связанные с оплатой.

Закладка Основное

Поле Переработчик может быть заполнено только тем партнером, который является Поставщиком, поскольку он будет оказывать услугу по переработке.
Заказ переработчику формируется с указанием договора, который будет определять порядок расчетов и выступать аналитикой учета товаров, переданных переработчику.

Для возможности поступления продукции на разные склады в рамках одного заказа, необходимо включить функциональную опцию Поступление товаров на несколько складов в разделе НСИ и администрирование - Закупки - Документы закупок.

Если заказ содержит различную продукцию, то может возникнуть необходимость в группировке затрат на эту продукцию:

  • Без группировки - затраты будут распределены на всю продукцию заказа;
  • По продукции - затраты будут распределены на продукцию, к которой они относятся. Группировка доступна, если в качестве Варианта обособления в передаче переработчику не выбрано значение По продукции;
  • По продукции и назначениям - затраты будут распределены на продукцию, к которой они относятся с учетом Назначений. Группировка доступна, если в качестве Варианта обособления в передаче переработчику выбрано значение По продукции;
  • По спецификациям продукции - затраты будут распределены на продукцию, которая относится к указанной спецификации. Группировка доступна, если в качестве Варианта обособления в передаче переработчику не выбрано значение По продукции;
  • По спецификациям и назначениям продукции - затраты будут распределены на продукцию, которая относится к указанной спецификации и назначению. Группировка доступна, если в качестве Варианта обособления в передаче переработчику выбрано значение По продукции;
  • По этапам производства - затраты будут распределены на продукцию, которая производится на последнем этапе цепочки к которой относится выбранный этап.

Закладка Продукция

Если заказ находится в статусе К исполнению или выше, становится обязательным поле Дата поступления. Его можно указать как для всего документа в целом, так и для каждой отдельной строки.

Для каждой строки табличных частей заполняется номенклатурная позиция, которая будет произведена из переданного сырья и материалов. Для удобства заполнения табличной части, можно использовать Подбор товаров.

Список Побочный и промежуточный выпуск по фиксированной стоимости предназначен для дополнительных выходных изделий, стоимость которых фиксирована и определяется в момент выпуска продукции. Такие изделия уменьшают себестоимость продукции на указанную стоимость.

Так же в табличной части возвратных отходов указывается Статья калькуляции для последующего заполнения статьи калькуляции в отчете переработчика.

Изделия могут быть списаны на расходы. Для этого необходимо установить соответствующий флаг и выбрать статью расходов с аналитикой расходов.

Закладка Материалы и работы

В табличной части указываются материалы и работы, которые наша организация готова предоставить переработчику.
Также может быть заполнена Залоговая стоимость передаваемых материалов.
Склад отгрузки может указываться отдельно в каждой строке (для удобства заполнения, можно выделить несколько строк, использовать команду Заполнить - Склад, выбрать склад и нажать Выбрать).

Так же в табличной части указывается Статья калькуляции для последующего заполнения статьи калькуляции в Отчете переработчика.

Если использовать собственное сырье не планируется и производство продукции на стороне планируется полностью из сырья и материалов переработчика, то в этом случае следует использовать Заказ поставщику.

Для строк с указанным действиями обособленного обеспечения или отгрузки, Назначение выбирается в соответствии с заданным в настройках вариантом обособления сырья в передаче переработчику.
Для варианта по направлению деятельности будет использоваться указанное направление деятельности.
Для варианта обособления по переработчику/договору в качестве Назначения будет выбран указанный Договор.
Для варианта по назначению продукции будет использоваться целевое назначение продкуции.
Для варианта по заказу переработчику будет использоваться сам заказ переработчику.
Данное назначение используется при передаче товаров переработчику в качестве назначения на складе отгрузки.

Закладка Услуги по переработке

На закладке указываются наименования услуг по переработке, количество, цена, сумма, ставка НДС и сумма НДС услуг, а так же статьи калькуляции, по которым услуги включаются в себестоимость выпущенных переработчиком изделий.

Закладка Дополнительно

На закладке указывается Валюта, Цена включает НДС, Налогооблажение НДС, Выпуск под деятельность и Подразделение, в рамках которого будут собираться все затраты по переработке переданного сырья.

Корректировка заказа

В процессе исполнения заказа можно изменять действия по обеспечению и отгрузке заказа непосредственно в строке списка материалов и работ, либо перезаполняя обеспечение. Также можно отменить исполнение строки заказа, установив флаг Отменено.

Для автоматической отмены всех недопоставленных изделий необходимо использовать команду Отменить непоставленные строки. Аналогично, для автоматической отмены всех неотгруженных материалов необходимо использовать команду Отменить неотработанные строки. Данные команды удобно использовать для закрытия заказа перед установкой статуса Закрыт. В случае если установлен запрет на закрытие частично отработанных заказов (функциональная опция Не закрывать частично отработанные заказы переработчикам в разделе НСИ и администрирование - Производство - Переработка на стороне), при проведении заказа переработчику в статусе Закрыт осуществляется контроль отгрузки и приемки по заказу: строки изделий заказа должны быть приняты, либо отменены, а строки материалов должны быть отгружены, либо отменены.

Статусы и приоритеты

Доступность статусов определяется функциональной опцией Статусы заказов переработчикам в разделе НСИ и администрирование - Производство - Переработка на стороне.
Для указания состояния документа предусмотрены следующие статусы:

  • на согласовании - заказ сформирован, но еще не согласован с переработчиком;
  • к исполнению - ожидается передача сырья и материалов переработчику для начала производства. Параллельно считается, что наша организация готова принимать выпущенную продукцию от переработчика. В этом статусе информация о потребностях попадает в рабочие места к оформлению Передач сырья переработчику, Отчетов переработчиков, Поступлений от переработчика;
  • закрыт - сформирован Отчет переработчика.

При работе с заказами можно установить приоритетность выполнения заказов. В дальнейшем пользователь может сортировать заказы в зависимости от их приоритета.
