DMService web service

This web service is used for synchronous integration of 1C:Document Management with other applications on the 1C:Enterprise platform as well as with other applications. Synchronous integration is applied for both implementation of the user interface for functionality of 1C:Document Management on behalf of integrated applications and for non-interactive interaction of integrated applications and applications with 1C:Document Management.

Examples of integration can be found in configuration "Library of integration with 1C:Document Management" (LID) supplied as a component of the distribution package.

Differences of PROF, CORP and DGU versions

The applied functionality absent in the PROF version, e.g., the built-in mail, is not supported by the web service of the PROF version.

Moreover, the web service of the PROF version does not support the automatic background update of the connected objects' attributes (queries DMGetChangesRequest, DMPutChangesRequest).

Functionality of services in CORP and DGU versions is identical.

The web service supports the single operation.

Web service DMMessageService

This web service is used for asynchronous integration of the Document Management with other applications and configurations according to the principle "sent and forgotten". The consumer applications sends requests for processing to the Document Management via calling the operation put and gets responses by calling the operation get. Messages may be processed by the Document Management in any order. Several responses may be generated for one query. This web service is convenient to use for integration of the Document Management with other applications on the ESB (Enterprise Service Bus) platform.

Operation: get

Returns: DMMessage

Parameter Type Description
queue string Name of the message queue, which are incoming in relation to the application integrated with 1C:Document Management. Before calling this operation, the queue must be created and configured in 1C:Document Management by the administrator. For more information about setting up queues see the 1C:Document Management configuration description.

Extracts one new message from the queue specified in the Queue parameter and returns it as an object of DMMessage class. If there are no new messages, then the DMOK class object is returned. If an error occurred when performing this operation, for example, the queue is incorrectly specified, then an object of DMERROR class is returned. Example of calling this operation in 1C:Enterprise:

DMApprovalType class

Base class: DMObject

Describes the option for arrangement of approval in the business process "Approval" in the Document Management. The ObjectID.ID property does not contain a GUID, but a text value corresponding to one of the values of the "Approval options" enumeration in the Document management.

Base class: DMObject
