Вид бюджета представляет собой иерархическую структуру объединенных в группы статьей и показателей бюджетов, а также их аналитик. Существует возможность использования в структуре бюджета нефинансовых показателей для расчета зависимых статей.

Вид бюджета используется для следующих целей:

  • табличного ввода плана в документе Экземпляр бюджета;
  • построения бюджетных отчетов требуемой структуры.

Виды бюджетов для ввода плановых данных


Ввод плановых данных в бюджеты осуществляется через документ Экземпляр бюджета.

Виды бюджетов, используемые для ввода плановых значений, должны иметь отметку Использовать для ввода плана. Такие бюджеты имеют ограничения по гибкости настройки структуры, в сравнении с видами бюджетов для отчетов.
Виды бюджетов без отметки использования для ввода плана используются только как отчеты, в них так же можно настроить вывод по сценариям и группировки данных по организациям и подразделениям.

Существует возможность добавления и удаления дополнительных аналитик в заданную Видом бюджета структуру аналитик во время ввода данных в документе Экземпляр бюджета , при этом предопределенные аналитики не доступны для удаления. Для добавления значений аналитик в экземпляре бюджета при перетаскивании аналитики в структуру бюджета нужно установить флаг Добавлять значения в экземпляре бюджета.

План-фактный анализ

Для построения отчетов План-План, План-Факт, План-План-Факт для сравнения разных сценариев между собой должны использоваться виды бюджетов, у которых снят флаг Использовать для ввода плана. Когда флаг Использовать для ввода плана снят в доступных элементах отчета в измерениях появляется элемент отчета Сценарий, с помощью которого можно выводить разные данные в рамках одного отчета.

Настройка отчетов план-факт и план-план с расчетом отклонения в суммовом или процентом отношении достигается указанием в группе, подчиненной периоду, двух сценариев, например, Месячный целевой и Фактические данные, и формулы по группе с типом Отклонение.

Ввод фактических данных в бюджетировании не требуется, фактические данные получаются по настроенным правилам получения фактических данных в процессе формирования отчетов или заполнения планов по фактическим данным предыдущих периодов.

Скользящее планирование

В виде бюджета можно явно определить способ составления бюджета: на фиксированный период или скользящий. И задать способ определения границы фактических данных.

При выборе вида бюджета Скользящий (способ планирования) при формировании отчетов и вводе Экземпляра бюджета нужно будет указывать только одну дату - от которой составляется бюджет. Дату окончания система рассчитает автоматически, исходя из настроек Вида бюджета.

Для составления планов на основе прогнозных данных на будущие периоды (например, составление бюджетов на год в сентябре, когда нет всего факта) используется граница фактических данных. В этом случае граница указывается с началом за 3 месяца до начала периода планирования (для годового бюджета это будет конец сентября).

Ввод бюджетов по объектам учета

С помощью гиперссылки в поле Аналитические разрезы бюджета: можно указать виды аналитик, которые указываются для всего документа экземпляр бюджета.

Пример. Если в качестве аналитического разреза бюджета выбрать вид аналитики "Контрагенты", то можно составить сводные бюджеты продаж по каждому контрагенту в отдельном документе Экземпляр бюджета. При работе со списком экземпляров бюджета если установить отбор по виду бюджета, то контрагент будет отображаться в списке.

Дополнительные настройки экземпляров бюджетов


Создание нового документа планирования возможно в режиме табличного документа или в режиме редактирования списком. Это определяется значением реквизита Основной способ ввода. По умолчанию предлагается ввод в форме табличного документа.

Для блокирования изменения ширины столбцов табличного документа экземпляра бюджета используется флаг Включить защиту табличного документа. По умолчанию защита табличного документа включена.

Для ускорения работы с бюджетами, которые содержат сложные структуры и большое количество данных возможно производить кэширование данных табличного документа экземпляра бюджета. Для этого используется флаг Сохранять бюджет в табличной форме экземпляра бюджета. По умолчанию кэширование отключено.

Для ускорения работы с бюджетами, которые содержат сложные структуры и большое количество данных запись данных экземпляра бюджета можно проводить отложено. Для этого предусмотрен флаг Проводить экземпляры бюджетов отложено. По умолчанию отложенное проведение не используется.

Настройка структуры бюджета

Структура бюджета формируется на одноименной закладке элемента справочника

Формирование структуры Вида бюджета осуществляется добавлением в нее элементов оформления, таблиц, группировок, формул, статей и показателей.
Измерениями отчетов являются: периоды, сценарии, организации, подразделения и другие виды аналитики.

Возможны различные комбинации подчинения аналитик и статей друг другу, как в колонках, так и строках таблиц. Например, допускается подчинение статей периоду или аналитикам и наоборот, комбинирование вложенности аналитик, периодов и статей.
Аналитика по статьям бюджетов может быть задана конечным списком значений или списком всего справочника, где были обороты или остатки по статье отчета.

При подборе конечных значений аналитики в структуру добавляется строка Прочее. Это позволяет конкретизировать планы по значимым значениям аналитик, а остальное вводить суммарно. Либо такую аналитику можно удалить и вводить планы только по указанным значениям.

Таблицы в Виде бюджетов могут быть трех видов:

  • Статьи и показатели в строках, аналитика или периоды в колонках;
  • Статьи и показатели в колонках, аналитика и периоды в строках;
  • Сложная таблица с произвольным расположением показателей и формул, в которой статьи и показатели задаются по конкретным ячейкам отчета.


Простая таблица в качестве объектов структуры содержит учетные объекты - статьи и показатели, а также аналитики. Сложная таблица в настройках структуры может содержать только аналитики, объекты учета задаются в настройках ячеек сложной таблицы.


Отметка Обратный знак в статье бюджета указывает на то, что обороты по данной статье будут вычитаться при получении итога по группе.

Настройка возможности количественного и валютного учета осуществляется в аналитике бюджетирования и включается в статьях и показателях бюджетов.

Далее по статьям и показателям в структуре Видов бюджетов настраивается количественный и\или суммовой режим отображения и ввода данных.

Перечень операций (оборотов по статьям бюджета или остатков по показателям), включаемых в тот или иной бюджет, определяется пользователем.

Пример. В БДДС включаются данные по операциям, изменяющим остаток денежных средств, в БДР – операции, влияющие на прибыль предприятия, а остаточные показатели формируют структуру прогнозного баланса.

Для оптимизации работы с бюджетами, содержащими сложные структуры и оперирующие большими массивами данных предусмотрен отчет, предлагающий рекомендации по ускорению расчета бюджетных формул. Отчет находится: Еще - Рекомендации по повышению скорости расчета формул при редактировании бюджета

Ограничение доступа к данным

В целях организации ограничений доступа пользователей к данным бюджетирования необходимо в настройках вида бюджета указывать поле Группа доступа

Стратегия ограничений прав доступа для формирования бюджетного отчета

При формировании бюджетного отчета возможна ситуация, когда часть данных запрещена к выводу, так как ограничена правами доступа. В этой связи в целях поддержания целостности формируемых данных существует две настройки, которые определяются полем При отсутствии полного доступа к необходимым данным:

  • Формировать с неполными данными - бюджет будет сформирован, данные будут ограничены в соответствии с ограничениями пользователя (например, доступны данные планируемых продаж только разрешенных организаций);
  • Не формировать - бюджетный отчет не будет формироваться, если в нем присутствуют данные, на которые наложены ограничения (например, при формировании баланса по холдингу в целом - если данные некоторых организаций недоступны, то баланс холдинга также будет недоступен).

Стратегия ограничений прав доступа для операндов формул в настройках бюджетной формы

В настройках операндов формул, которые задаются в настройках структуры бюджета, возможно указание одного из двух режимов получения данных для операнда формулы. Настройка указывается в разделе Настройки получения данных операнда:

  • Учитывать ограничения доступа к данным - в данный операнд будут выбраны только разрешенные данные;
  • Не учитывать ограничения доступа к данным - в данный операнд будут выбраны все данные, исключая систему ограничений доступа к данным. Такая настройка может использоваться при расчете относительных величин, например, когда требуется определить долю продаж разрешенных организаций в продажах группы компаний в целом.
