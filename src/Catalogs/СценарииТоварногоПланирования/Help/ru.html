Сценарии товарного планирования

Сценарий планирования хранит настройки процесса планирования: последовательность создания планов, периодичность планирования, разрезы планирования и т.п.
Для возможности работы со сценариями у пользователя должна быть установлена роль Добавление изменение сценариев планирования.

Основное
  • Периодичность (Год, полугодие, квартал, месяц и т.д.) определяет, в каких интервалах будут вводиться данные планов, создаваться заказы по планам и анализироваться план-факт. Периодичность Месяц означает, что для целей планирования не важно, 1-е число или 31-е число месяца указано в факте (исполнении плана).
  • Отображение периода (Диапазон дат или Номер периода в пределах года)
  • Планировать закупки (Только по количеству или По количеству и сумме) - позволяет вводит планы закупок с ценами и суммами или только количественные планы.
  • Планировать продажи (Только по количеству или По количеству и сумме) - аналогично настройке для плана закупок.
  • Валюта сценария, если отличается от управленческой валюты, то в отчетах по исполнению планов фактические данные будут пересчитаны в эту валюту.
  • Управление процессом планирования - для сценария можно описать процесс планирования. Процесс планирования – это последовательность этапов. Каждый этап планирует одно из трех: обеспечение, потребление или остатки. К этапам привязываются виды планов. Формировать планы можно последовательно по этапам через помощник Управление процессом планирования.
  • Планирование по назначениям - позволяет использовать в планах сквозную аналитику назначение (планировать в разрезе направлений деятельности, клиентских заказов и т.д.).

Совет. При использовании детализации планов по назначениям не рекомендуется выбирать данные из прошлых периодов или использовать смещение по периоду. Если в планы попадут назначения, которые уже обеспечены или реализованы ранее, чем период планирования, то обособленные потребности нельзя будет сопоставить при план-фактном анализе. Через рабочие места по формированию заказов по планам будут оформлены заказы сверх потребностей из заказов, если включена опция Склад и доставка - Обеспечение потребностей - Разрешить обособление товаров сверх потребности.

Настройки планирования продаж по товарным категориям

Расчет скорости продаж - позволяет получить план продаж по номенклатуре из плана продаж по товарным категориям с учетом скорости продаж и рейтингов номенклатуры. Доступно, если включена функциональная опция Рейтинги продаж номенклатуры в разделе НСИ и администрирование - CRM и маркетинг - Маркетинг.

Настройки планирования производства

Если установлен флаг Расчет потребностей в материалах, видах РЦ и трудовых ресурсах, то в плане производства нужно будет ввести ресурсную спецификацию для каждой строки номенклатуры. После проведения документа плана производства, система запустит задание на расчет ресурсов, необходимых для выполнения плана (материалы, трудовые ресурсы, время видов рабочих центров). Длительность расчета зависит от структуры изделий указанных в спецификациях. Если отключить флаг, то колонка спецификации в плане скроется и расчет производиться не будет.

При расчете материалов под план можно учесть вероятность применения материалов. Предусмотрено три стратегии расчета: Наиболее вероятная, Оптимистичная или Пессимистичная. Выбранная стратегия влияет на способ учета вероятности применения материала, заданной в спецификации, и на необходимость планирования альтернативных материалов.

Для расчета дат потребностей в ресурсах и дат запуска/выпуска полуфабрикатов в сценарии может быть задан Календарь работы, отличающийся от основного календаря предприятия. Это позволяет моделировать работу предприятия в разных сценария по разным календарям.

Структура планов

В сценарии можно описать процесс планирования. Процесс планирования – это последовательность этапов. Каждый этап планирует одно из трех: обеспечение, потребление или остатки.
К этапам можно привязать виды планов. В них указывается что конкретно мы планируем: продажи по категориям, продажи по номенклатуре, внутренние потребления, производство, закупки, сборку/разборку, остатки.
Планы доступны, если включены соответствующие опции в разделе НСИ и администрирование - Бюджетирование и планирование - Планирование.

Процесс планирования состоит из четырех этапов: ввод начальных остатков (планирование остатков), планирование продаж (планирование потребностей), планирование производства (планирование обеспечения), планирование закупок (планирование обеспечения).
Для каждого этапа вводятся свои виды планов. Например, для этапа "планирование продаж" введены два вида плана: первый для планирования продаж по категориям, второй для планирования продаж по номенклатуре.

Дополнительные настройки

Планы по Сценарию планирования могут отражаться в подсистеме Бюджетирование. Для этого необходимо установить флаг Отражать планы в бюджетировании и указать сценарий бюджетирования. Отражение в бюджетировании конкретного плана уточняется в виде плана.
